# 自适应伪标签阈值后续实验方案更新

生成时间：2026-06-30
依据结果：`20260627_pseudo_15-5_overlap_batchclass_q0p7_seed1_bs32`

## 1. 更新后的判断

首次 `15-5 overlap batch_class q0.7` 实验已经跑通，但当前证据还不能支撑"自适应伪标签阈值涨点"的结论。原因很简单：缺少同协议对照。

当前只知道：

```text
15-5 overlap, seed1, bs32, deeplabv3_resnet101, buffer8196, gamma1
step1, pseudo_label_strategy=batch_class, quantile=0.7
all mIoU = 70.45
old mIoU = 79.29
new mIoU = 42.16
```

但还不知道：

```text
同一个 step0 + 15-5 overlap + pseudo off 会是多少？
同一个 step0 + 15-5 overlap + fixed0.7 会是多少？
同一个 step0 + 15-5 overlap + batch_global q0.7 会是多少？
```

所以当前不应马上做更复杂代码改动。优先用最小实验矩阵把因果关系补齐。

## 2. 是否需要进阶代码改动

当前结论：暂不建议立刻做进阶训练代码改动。

理由：

1. 现在最大问题不是能力不足，而是证据链不完整。
2. 已有实现已经覆盖 `off/fixed/batch_global/batch_class/artifact_class` 的核心策略面。
3. 在没有 `off/fixed` 结果前，继续加 class balance、margin、artifact 校准等机制会让变量更多，结论更难归因。

可以准备但不立即执行的后续代码项：

| 代码项 | 目的 | 触发条件 |
| --- | --- | --- |
| raw-mask pseudo audit 工具 | 用原始未隐藏 GT 估计伪标签 precision/recall/confusion，只做诊断不进训练 | `batch_class` 与 baseline 差距不清晰，或类别 15 支配严重 |
| per-class cap / quota | 防止某个旧类伪标签数量支配闭式解 | 确认 class 15 或其它类伪标签污染结果 |
| offline artifact threshold | 让阈值脱离 batch 抖动，更可复现 | batch-level 方法相对 fixed 有收益但波动大 |
| margin threshold sweep | 同时要求 top1 和 top2 拉开差距 | fixed/adaptive 都有明显错误伪标签迹象 |

优先级最高的是 audit 工具，但它应该在补齐 baseline 之后再做。它不改变训练，只帮助解释伪标签质量，因此适合作为下一阶段分析增强项。

## 4. 判定标准

| 结果形态 | 结论 |
| --- | --- |
| `batch_class > fixed > off` | 自适应阈值有正向证据，可以推进 disjoint 和多步复验 |
| `fixed >= batch_class > off` | 伪标签有用，但类别级自适应暂不成立，检查 fallback 和 class imbalance |
| `off >= batch_class/fixed` | 伪标签在 `15-5 overlap` 可能无收益或有噪声，转向 raw-mask audit |
| `batch_global ≈ batch_class` | 类别级设计贡献弱，优先做 offline artifact 或 class cap |
| `batch_class old 涨但 new 跌` | 伪标签可能过度保护旧类，检查新类边界和背景污染 |

硬性要求：所有对比必须使用同一个 step0 checkpoint。否则 step1 差异无法归因到伪标签策略。
