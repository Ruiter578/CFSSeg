## 0. 2026-07-01 更新结论

截至 2026-07-01，Phase A 的两轮搜索已经完成：

1. 固定阈值搜索：`fixed0.6 / fixed0.8 / fixed0.9`。
2. 低 q 自适应搜索：`batch_global q=0.1/0.3/0.5`、`batch_class q=0.1/0.3/0.5`、`batch_class q=0.3 + min_conf=0.6`。

最新结果说明：

```text
fixed0.6 是当前 15-5 overlap step1 的最强结果；
batch_class q0.1 已经接近 fixed0.6，但没有超过；
继续扩大 batch-level quantile 网格的优先级不高；
下一步应该转向 offline artifact calibration 和 pseudo-label quality audit。
```

换句话说，当前结论不是“自适应伪标签失败”，而是：

```text
第一版 batch 内分位数自适应没有形成超过固定阈值的优势；
真正值得继续验证的是更稳定的离线类别阈值 artifact，以及质量感知策略。
```

## 1. 先用一句话讲清楚当前情况

第一轮 4 组实验说明：**伪标签有用，但当前“q=0.7 的自适应阈值”太保守，没有超过固定阈值 `fixed0.7`。**

也就是说，我们现在不能说“自适应伪标签阈值方法已经涨点”。更准确的说法是：

```text
固定阈值伪标签在 15-5 overlap 中有效；
batch_global / batch_class q0.7 跑通了，但不是好方法；
后续要重新设计“自适应”的定义，并系统搜索参数和策略。
```

结果如下：

### 2.1 这张表说明了什么

第一，`fixed0.7` 明显最好。它比不用伪标签高 `+0.4292` all mIoU。

第二，`batch_global q0.7` 基本没有效果。它只比不用伪标签高 `+0.0166` all mIoU，可以近似看作没提升。

第三，`batch_class q0.7` 比 `batch_global q0.7` 好一点，说明“按类别分配伪标签名额”有一点价值；但它仍然比 `fixed0.7` 低 `-0.2848` all mIoU。

所以第一轮的初步结论是：

```text
伪标签机制值得继续做；
但 q=0.7 不是一个好设置；
当前自适应方案需要重构，而不是直接包装成论文贡献。
```

第二，`batch_class q0.1` 基本追平 fixed baseline，但没有超过。它说明类别自适应不是完全无效，但当前“按 batch 取分位数”的实现还不足以成为方法亮点。

第三，`q` 越高，接受比例越低，结果越差。当前任务更需要恢复旧类伪标签召回，而不是继续提高置信度门槛。

因此后续不应继续盲目扩大 batch quantile 网格，而应进入更稳定、更可复现的 `artifact_class` 路线。

## 3. 为什么 q=0.7 太保守

这里的 `q=0.7` 指 quantile，也就是分位数。

假设有 100 个候选伪标签像素，每个像素都有一个 teacher 置信度。`q=0.7` 的意思是：

```text
找到 70% 分位点；
只保留分数高于这个分位点的像素；
大约只接受最高的 30%。
```

这和当前实验统计完全一致：

```text
batch_global q0.7 accepted ratio = 30.0003%
batch_class q0.7 accepted ratio  = 30.0008%
```

问题在于：当前 `15-5 overlap step1` 是最后一个增量 step，没有未来类再登场。此时背景里最主要的问题是旧类 `1-15` 被隐藏成 background。旧 teacher 对很多旧类像素其实很有把握，所以 `fixed0.7` 一口气接受了 90.35% 候选，反而恢复了更多正确旧类监督。

因此：

```text
q=0.7 的 top 30% 高精度策略，在当前任务里召回太低。
```

## 4. 后续“自适应”应该怎么重新定义

“自适应”不应该只等于“按 batch 取 70% 分位点”。后续至少有四种更合理的方向。

### 4.1 接受比例自适应

继续用 quantile，但不要只用 `q=0.7`。

| q | 大致含义 |
| ---: | --- |
| `0.1` | 接受 top 90%，接近 `fixed0.7` 的高召回 |
| `0.3` | 接受 top 70%，比较折中 |
| `0.5` | 接受 top 50% |
| `0.7` | 接受 top 30%，当前已证明太保守 |

后续要验证的是：低 q 的自适应能否保留 fixed 的高召回，同时比 fixed 更能过滤噪声。

### 4.2 固定阈值校准

现在 `fixed0.7` 是强 baseline，但它不一定是论文等价配置。

原论文写的是不确定性阈值：

```text
tau = 0.4
```

当前代码用的是置信度阈值。二者近似关系是：

```text
uncertainty = 1 - confidence
tau = 0.4  <=>  confidence = 0.6
```

所以后续必须补：

```text
fixed0.6
fixed0.8
fixed0.9
```

其中 `fixed0.6` 是 paper-equivalent baseline，`fixed0.8/0.9` 用来判断 `fixed0.7` 是否刚好最优。

### 4.3 离线校准 artifact

batch-level 阈值会随每个 batch 的样本组成波动。更稳的做法是：

1. 先用 teacher 扫一遍 step1 train set；
2. 统计每个旧类的候选置信度分布；
3. 生成一个 `threshold artifact`；
4. 正式训练时固定读取这个 artifact。

这样阈值不会因为 batch 顺序变化而变，实验更可复现，也更适合写论文。

### 4.4 质量感知自适应

只看 top1 confidence 还不够。后续可以加入：

- top1 和 top2 的差距，也就是 margin；
- 每个类别最多接受多少伪标签，避免大类支配；
- 每个类别最少保留多少伪标签，避免小类饥饿；
- confidence-weighted pseudo label，让不太确定的伪标签权重小一点；
- raw-mask audit，用真实完整标签评估伪标签质量后，再设计类别专属策略。

这些是更进取的方向，但需要先通过 Phase A 找到有希望的参数区间。

## 5. 以后要怎么和原论文 69.36 对比

现在的 `15-5 overlap` 结果不能直接和论文的 69.36 对比。

原因是：

| 项目 | 当前实验 | 论文 69.36 |
| --- | --- | --- |
| task | `15-5` | `15-1` |
| setting | `overlap` | `overlapped` |
| 增量步数 | 1 个 step | 5 个 step |
| 是否论文主表协议 | 否 | 是 |

所以要和 69.36 对比，必须跑：

```text
TASK=15-1
SETTING=overlap
START_STEP=0
END_STEP=5
```

并且至少包含：

```text
off
fixed0.6    # 论文 tau=0.4 的近似等价
fixed0.7    # 当前代码强 baseline
best adaptive
```

如果 `best adaptive` 同时超过本地 fixed baseline 和论文 `69.36`，才能说它在论文协议上有强正向证据。

## 7. 按优先级的工作流

### 阶段一：Phase A 搜索基础设施与 fixed sweep（已完成）

目标：先把实验矩阵跑起来，不再手写单条命令。

需要实现：

```text
tools/run_pseudo_label_grid.sh
tools/summarize_pseudo_label_grid.py
configs/pseudo_label_phaseA_fixed_sweep.tsv
```

第一组启动：

```text
fixed0.6
fixed0.8
fixed0.9
```

这组实验回答：

```text
fixed0.7 是否只是偶然强？
论文等价 fixed0.6 是否更强？
更高阈值 fixed0.8/0.9 是否能减少噪声并涨点？
```

当前答案：

```text
fixed0.6 最强；
fixed0.8 / fixed0.9 随阈值升高变差；
当前任务需要高召回，固定阈值 baseline 必须更新为 fixed0.6。
```

### 阶段二：Phase A adaptive low-q sweep（已完成）

在 fixed sweep 完成后，启动：

```text
batch_global q=0.1 / 0.3 / 0.5
batch_class  q=0.1 / 0.3 / 0.5
batch_class q=0.3 + min_conf=0.6
```

判断标准：

```text
必须超过 fixed0.6 的 70.7731，才算 adaptive 初步成立。
```

当前答案：

```text
batch_class q0.1 接近 fixed0.6，但没有超过；
batch_global q0.1 接近 fixed0.7，但也没有超过 fixed0.6；
q=0.3/0.5/0.7 都明显更弱；
batch 内 quantile 自适应暂时不能作为主方法。
```

### 阶段三：Phase B offline artifact calibration + raw-mask audit（下一步）

这一阶段是当前最重要的下一步。它对应第 4 章里的两类未充分验证的方法：

```text
4.3 离线校准 artifact；
4.4 质量感知自适应。
```

为什么不能直接继续启动训练：

```text
现有 grid runner 已能跑 fixed / batch_global / batch_class；
训练代码内部已经支持 artifact_class；
但 grid 配置还不能按行传入 threshold_artifact；
也缺少把 calibration artifact 和后续 artifact_class 训练串起来的执行文档。
```

因此阶段三先做轻量代码与工具链补齐：

```text
扩展 grid runner，使每行可以传入 threshold_artifact / threshold_max_batches；
新增 artifact calibration grid；
新增 artifact_class train grid；
用 tools/calibrate_pseudo_label_thresholds.py 生成离线类别阈值；
训练 artifact_class，并和 fixed0.6 / batch_class q0.1 比较；
实现或补齐 raw-mask audit，解释 fixed 与 artifact 的 precision / recall 差异。
```

建议优先验证的 artifact 参数：

| 组别 | quantile | 目的 |
| --- | ---: | --- |
| `artifact q0.05` | 0.05 | 接受约 top 95%，对齐 `fixed0.6` 的高召回 |
| `artifact q0.10` | 0.10 | 接受约 top 90%，对齐 `batch_class q0.1` |
| `artifact q0.20` | 0.20 | 接受约 top 80%，观察稍高精度是否更稳 |

阶段三的判定标准：

```text
如果 artifact_class > fixed0.6：
    自适应阈值线重新具备主方法潜力。
如果 artifact_class 只接近 fixed0.6：
    需要 raw-mask audit 判断是否能通过 margin、类别 cap、权重伪标签继续提升。
如果 artifact_class 明显低于 fixed0.6：
    伪标签线应降级为消融或辅助机制，主线资源转向 RHL / 集成。
```

### 阶段四：15-5 disjoint 复验

只带 top 策略过去：

```text
off
fixed0.6
best adaptive from Phase B
```

如果 overlap 有效但 disjoint 无效，需要重新解释方法适用边界。

### 阶段五：15-1 / 10-1 overlapped 论文对齐

这是论文主表对齐阶段。至少跑：

```text
off
fixed0.6
fixed0.7
best adaptive from Phase B
```

最后判断：

```text
best adaptive > fixed0.6 且 > 69.36：
    可以作为论文方法主证据。
best adaptive > fixed0.6 但 <= 69.36：
    本地有效，但还不能说超过原论文。
best adaptive <= fixed0.6：
    adaptive 线降级为消融，主线转回 RHL / 集成。
```

`10-1 overlapped` 不一定立刻跑全矩阵。更合理的做法是只带 `15-1` 中最有希望的两组过去做压力测试。

### 阶段映射总结

第 4 章的四种方向在新版工作流中的位置如下：

| 第 4 章方向 | 对应阶段 | 当前状态 |
| --- | --- | --- |
| 接受比例自适应 | 阶段二 | 已完成，`batch_class q0.1` 接近但未超过 `fixed0.6` |
| 固定阈值校准 | 阶段一 | 已完成，最佳 baseline 更新为 `fixed0.6` |
| 离线校准 artifact | 阶段三 | 下一步重点，需要补齐 grid/artifact 执行链 |
| 质量感知自适应 | 阶段三之后 | 先由 raw-mask audit 提供证据，再决定 margin/cap/weighted 是否值得实现 |

## 8. 后续实验总结口径

以后每轮实验都要按下面格式总结：

| 字段 | 必须说明 |
| --- | --- |
| 协议 | task、setting、step、是否论文主表协议 |

## 9. 当前阶段最终判断

当前自适应伪标签阈值线还没有失败，但 batch-level quantile 这一版已经不足以支撑主方法。

后续要做的不是继续扩大 `q` 值表，而是：

```text
把 best fixed baseline 固定为 fixed0.6；
用 offline artifact 校准更稳定的类别阈值；
用 audit 工具解释 fixed / batch / artifact 的伪标签质量差异；
再决定是否上 15-5 disjoint 和 15-1 / 10-1 overlapped 论文协议；
最后再决定它是否能成为论文方法贡献。
```
