# 自适应伪标签阈值二阶段方案与小白版工作流

生成时间：2026-06-30
适用分支：`feature/adaptive-pseudo-label`
面向读者：第一次接触 SegACIL / CFSSeg 伪标签实验的人

## 1. 先用一句话讲清楚当前情况

第一轮 4 组实验说明：**伪标签有用，但当前“q=0.7 的自适应阈值”太保守，没有超过固定阈值 `fixed0.7`。**

也就是说，我们现在不能说“自适应伪标签阈值方法已经涨点”。更准确的说法是：

```text
固定阈值伪标签在 15-5 overlap 中有效；
batch_global / batch_class q0.7 跑通了，但不是好方法；
后续要重新设计“自适应”的定义，并系统搜索参数和策略。
```

## 2. 第一轮 4 组实验结果总结

这 4 组实验都在同一个协议下完成：

```text
dataset: VOC 2012
task: 15-5
setting: overlap
step: 1
model: deeplabv3_resnet101
batch size: 32
buffer: 8196
gamma: 1
seed: 1
step0 checkpoint: 20260627_pseudo_15-5_overlap_batchclass_q0p7_seed1_bs32
```

结果如下：

| 策略 | all mIoU | old `0-15` | new `16-20` | 接受伪标签比例 |
| --- | ---: | ---: | ---: | ---: |
| 不用伪标签 `off` | 70.3091 | 79.1179 | 42.1209 | - |
| 全局自适应 `batch_global q0.7` | 70.3257 | 79.1423 | 42.1124 | 30.0003% |
| 类别自适应 `batch_class q0.7` | 70.4535 | 79.2941 | 42.1636 | 30.0008% |
| 固定阈值 `fixed0.7` | **70.7383** | **79.6346** | **42.2703** | **90.3503%** |

### 2.1 这张表说明了什么

第一，`fixed0.7` 明显最好。它比不用伪标签高 `+0.4292` all mIoU。

第二，`batch_global q0.7` 基本没有效果。它只比不用伪标签高 `+0.0166` all mIoU，可以近似看作没提升。

第三，`batch_class q0.7` 比 `batch_global q0.7` 好一点，说明“按类别分配伪标签名额”有一点价值；但它仍然比 `fixed0.7` 低 `-0.2848` all mIoU。

所以第一轮的初步结论是：

```text
伪标签机制值得继续做；
但 q=0.7 不是一个好设置；
当前自适应方案需要重构，而不是直接包装成论文贡献。
```

## 3. 为什么 q=0.7 太保守

这里的 `q=0.7` 指 quantile，也就是分位数。

假设有 100 个候选伪标签像素，每个像素都有一个 teacher 置信度。`q=0.7` 的意思是：

```text
找到 70% 分位点；
只保留分数高于这个分位点的像素；
大约只接受最高的 30%。
```

这和当前实验统计完全一致：

```text
batch_global q0.7 accepted ratio = 30.0003%
batch_class q0.7 accepted ratio  = 30.0008%
```

问题在于：当前 `15-5 overlap step1` 是最后一个增量 step，没有未来类再登场。此时背景里最主要的问题是旧类 `1-15` 被隐藏成 background。旧 teacher 对很多旧类像素其实很有把握，所以 `fixed0.7` 一口气接受了 90.35% 候选，反而恢复了更多正确旧类监督。

因此：

```text
q=0.7 的 top 30% 高精度策略，在当前任务里召回太低。
```

## 4. 后续“自适应”应该怎么重新定义

“自适应”不应该只等于“按 batch 取 70% 分位点”。后续至少有四种更合理的方向。

### 4.1 接受比例自适应

继续用 quantile，但不要只用 `q=0.7`。

| q | 大致含义 |
| ---: | --- |
| `0.1` | 接受 top 90%，接近 `fixed0.7` 的高召回 |
| `0.3` | 接受 top 70%，比较折中 |
| `0.5` | 接受 top 50% |
| `0.7` | 接受 top 30%，当前已证明太保守 |

后续要验证的是：低 q 的自适应能否保留 fixed 的高召回，同时比 fixed 更能过滤噪声。

### 4.2 固定阈值校准

现在 `fixed0.7` 是强 baseline，但它不一定是论文等价配置。

原论文写的是不确定性阈值：

```text
tau = 0.4
```

当前代码用的是置信度阈值。二者近似关系是：

```text
uncertainty = 1 - confidence
tau = 0.4  <=>  confidence = 0.6
```

所以后续必须补：

```text
fixed0.6
fixed0.8
fixed0.9
```

其中 `fixed0.6` 是 paper-equivalent baseline，`fixed0.8/0.9` 用来判断 `fixed0.7` 是否刚好最优。

### 4.3 离线校准 artifact

batch-level 阈值会随每个 batch 的样本组成波动。更稳的做法是：

1. 先用 teacher 扫一遍 step1 train set；
2. 统计每个旧类的候选置信度分布；
3. 生成一个 `threshold artifact`；
4. 正式训练时固定读取这个 artifact。

这样阈值不会因为 batch 顺序变化而变，实验更可复现，也更适合写论文。

### 4.4 质量感知自适应

只看 top1 confidence 还不够。后续可以加入：

- top1 和 top2 的差距，也就是 margin；
- 每个类别最多接受多少伪标签，避免大类支配；
- 每个类别最少保留多少伪标签，避免小类饥饿；
- confidence-weighted pseudo label，让不太确定的伪标签权重小一点；
- raw-mask audit，用真实完整标签评估伪标签质量后，再设计类别专属策略。

这些是更进取的方向，但需要先通过 Phase A 找到有希望的参数区间。

## 5. 以后要怎么和原论文 69.36 对比

现在的 `15-5 overlap` 结果不能直接和论文的 69.36 对比。

原因是：

| 项目 | 当前实验 | 论文 69.36 |
| --- | --- | --- |
| task | `15-5` | `15-1` |
| setting | `overlap` | `overlapped` |
| 增量步数 | 1 个 step | 5 个 step |
| 是否论文主表协议 | 否 | 是 |

所以要和 69.36 对比，必须跑：

```text
TASK=15-1
SETTING=overlap
START_STEP=0
END_STEP=5
```

并且至少包含：

```text
off
fixed0.6    # 论文 tau=0.4 的近似等价
fixed0.7    # 当前代码强 baseline
best adaptive
```

如果 `best adaptive` 同时超过本地 fixed baseline 和论文 `69.36`，才能说它在论文协议上有强正向证据。

## 6. 总体路线：三条线并行

后续不要只跑一个最小实验。应该分三条线：

### 6.1 保守验证线

目标：确认固定阈值 baseline 到底有多强。

实验：

```text
fixed0.6
fixed0.7
fixed0.8
fixed0.9
```

### 6.2 进取探索线

目标：主动搜索更强的自适应策略。

实验：

```text
batch_global q=0.1 / 0.3 / 0.5
batch_class  q=0.1 / 0.3 / 0.5
batch_class q=0.3 + min_conf=0.6
```

### 6.3 论文证据线

目标：最终和原论文主表对齐。

实验：

```text
15-5 disjoint
15-1 overlapped
10-1 overlapped
```

## 7. 按优先级的工作流

### 阶段一：Phase A 搜索基础设施与 fixed sweep

目标：先把实验矩阵跑起来，不再手写单条命令。

需要实现：

```text
tools/run_pseudo_label_grid.sh
tools/summarize_pseudo_label_grid.py
configs/pseudo_label_phaseA_fixed_sweep.tsv
```

第一组启动：

```text
fixed0.6
fixed0.8
fixed0.9
```

这组实验回答：

```text
fixed0.7 是否只是偶然强？
论文等价 fixed0.6 是否更强？
更高阈值 fixed0.8/0.9 是否能减少噪声并涨点？
```

### 阶段二：Phase A adaptive sweep

在 fixed sweep 完成后，启动：

```text
batch_global q=0.1 / 0.3 / 0.5
batch_class  q=0.1 / 0.3 / 0.5
batch_class q=0.3 + min_conf=0.6
```

判断标准：

```text
必须超过 fixed0.7 的 70.7383，才算 adaptive 初步成立。
```

### 阶段三：raw-mask audit

实现并运行：

```text
tools/audit_pseudo_label_quality.py
```

要回答：

```text
fixed0.7 为什么强？
低 q adaptive 是高 precision 还是高 recall？
哪些类别的伪标签污染最严重？
class 9 / 11 / 15 的提升来自哪些像素？
```

### 阶段四：15-5 disjoint 复验

只带 top2 策略过去：

```text
off
best fixed
best adaptive
```

如果 overlap 有效但 disjoint 无效，需要重新解释方法适用边界。

### 阶段五：15-1 overlapped 对比 69.36

这是论文主表对齐阶段。至少跑：

```text
off
fixed0.6
fixed0.7
best adaptive
```

最后判断：

```text
best adaptive > local fixed baseline 且 > 69.36：
    可以作为论文方法主证据。
best adaptive > local fixed baseline 但 <= 69.36：
    本地有效，但还不能说超过原论文。
best adaptive <= fixed baseline：
    adaptive 线降级为消融，主线转回 RHL / 集成。
```

## 8. 后续实验总结口径

以后每轮实验都要按下面格式总结：

| 字段 | 必须说明 |
| --- | --- |
| 协议 | task、setting、step、是否论文主表协议 |
| base checkpoint | path 与 sha256 |
| 策略 | off / fixed / batch_global / batch_class / artifact |
| 参数 | confidence、quantile、min_conf、margin |
| 指标 | all / old / new mIoU |
| 伪标签统计 | candidate、accepted、accepted ratio、per-class accepted |
| 结论边界 | 是本地同协议结论，还是论文协议结论 |
| 下一步 | 继续、淘汰、复验、上论文协议 |

## 9. 当前阶段最终判断

当前自适应伪标签阈值线还没有失败，但第一版 `q=0.7` 已经失败。

后续要做的不是继续证明 `q=0.7`，而是：

```text
用 grid runner 系统搜索阈值空间；
用 audit 工具解释伪标签质量；
用 15-1 / 10-1 overlapped 对齐原论文；
最后再决定它是否能成为论文方法贡献。
```
