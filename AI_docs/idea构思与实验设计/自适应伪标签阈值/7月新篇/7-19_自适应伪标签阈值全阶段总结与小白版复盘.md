## 0. 如果只看这一节

### 0.1 一句话结论

这条研究线已经证明了两件不同的事：

1. **伪标签本身值得保留。** 在旧类会被当前标注隐藏成背景的 `overlap/disjoint` 协议中，把旧类像素从背景中“捞回来”能稳定改善结果。
2. **“类别特异的硬阈值”不值得继续主攻。** 当总体阈值和接收规模被 matched-global 实验控制后，`artifact_class` 六组平均只比全局阈值高 `+0.002005 pp`，远低于预注册的 `+0.1 pp` 实际意义门槛，而且在 disjoint 三组中全部轻微反向。

因此，当前路线不是“伪标签失败”，也不是“继续把 q 调得更细”，而是：

> 保留高覆盖伪标签，停止 hard classwise threshold 微调，把 teacher 的 confidence/margin 以连续权重传入 C-RLS 解析目标。

## 1. 小白先理解：这个方法到底在解决什么

### 1.1 什么是类增量语义分割

语义分割要给每个像素分配类别。类增量学习则把类别分阶段交给模型：

- step0 学旧类；
- step1 再加入新类；
- 最后要求模型同时识别新旧所有类别。

VOC `15-5` 的含义是：

- step0：背景加 15 个旧前景类；
- step1：一次加入 5 个新前景类；
- 最终评估 21 类。

### 1.2 为什么旧类会被“背景吞掉”

在 `overlap/disjoint` 的当前 step 标注中，图片里虽然可能存在旧类物体，但这些旧类像素不一定继续保留旧类标签，常被写成背景 `0`。

模型如果直接用这种标签学习，就会收到错误监督：

```text
真实旧类像素 → 当前标签写成 background → 解析分类头被要求把旧类推向背景
```

这就是这里所说的 semantic drift。

### 1.3 伪标签做了什么

旧模型在 step0 已经学过旧类，因此可以当 teacher。当前方法只检查被标成背景的像素：

1. teacher 给出预测类别和 confidence；
2. 如果 teacher 预测为旧类，就把它列为候选；
3. 如果候选通过阈值，就把背景标签改回 teacher 的旧类预测；
4. 新类真值、可见旧类真值和 ignore 标签不被覆盖。

伪标签的目标不是给新类造标签，而是恢复被当前协议隐藏的旧类监督。

### 1.4 为什么阈值是研究问题

固定阈值如 `0.7` 很直观，但不同旧类的 teacher confidence 分布可能不同：

- 容易类别的正确预测可能普遍在 `0.9` 以上；
- 困难类别的正确预测可能只有 `0.4–0.6`；
- 同一个 `0.7` 可能让容易类混入噪声，也可能让困难类几乎没有伪标签。

最初的研究假设因此是：

> 每个旧类根据自己的 confidence 分布设置阈值，可能比所有类别共用固定阈值更好。

后续实验正是一步步检验并最终修正了这个假设。

## 2. 三种 setting 的边界

| setting | 当前 step 中旧类标签情况 | 伪标签价值 | 当前代码行为 |
| --- | --- | --- | --- |
| `sequential` | 旧类标签通常仍可见 | 不是主要矛盾 | 启用伪标签会显式报错 |
| `disjoint` | 图片选择更严格，旧类常被隐藏成背景 | 高 | 支持 |
| `overlap` | 当前类图片中可混有旧类/未来类，非当前目标被映射为背景 | 高 | 支持 |

因此，本文所有伪标签机制结论只适用于 `overlap/disjoint`，不能外推成 sequential 结论。

历史 `15-1 overlap` 结果用于说明伪标签在多步链路中的方向性价值；它并不证明类别特异阈值有效。

## 3. 方法演化时间线

## 3.1 阶段 0：原始固定阈值

最早的思路是使用固定 confidence：

\[
\hat y_i=
\begin{cases}
\operatorname{teacher}(x_i), & y_i=0,\ \hat c_i\in C_{\text{old}},\ p_i\ge \tau,\\
y_i, & \text{otherwise}.
\end{cases}
\]

其中：

- \(y_i=0\)：当前标签是背景；
- \(\hat c_i\)：teacher top-1 类别；
- \(C_{\text{old}}\)：旧前景类集合；
- \(p_i\)：teacher top-1 confidence；
- \(\tau\)：固定阈值，如 0.7。

优点是简单，缺点是所有类别共用同一个门槛。

## 3.2 阶段 1：batch 内 class-wise quantile 构思

最初的“自适应阈值”定义为：

1. 在当前 batch 内收集每个旧类的候选 confidence；
2. 对类别 \(c\) 取 q 分位数作为阈值 \(\tau_c\)；
3. 只接受 \(p_i\ge\tau_{c_i}\) 的候选。

例如 `q=0.7` 表示阈值位于 70% 分位点，因此大约只接收 confidence 最高的 30% 候选。

早期规划的最小策略包括：

- `fixed`；
- `batch_quantile`；
- `class_quantile`；
- `quantile=0.7`；
- `min_conf=0.5`；
- `max_conf=0.95`；
- `min_pixels=64`。

这一阶段只是构思，尚未完成真实代码路径和协议审查。

## 3.3 阶段 2：代码与协议全面审计

正式实现前发现了三个关键问题。

### 问题一：旧路径可能根本没有执行

早期条件使用过类似 `curr_step > 1` 的 guard，导致 VOC `15-5` 只有 step1 时伪标签不会真正执行。

这意味着“命令里写了 use pseudo-label”不等于机制真的进入训练。

### 问题二：teacher 输出格式不统一

- DeepLab teacher 可能返回 tuple，logits 为 NCHW；
- AIR teacher 可能返回 tensor，布局为 NHWC；
- BCE 使用 sigmoid，CE/focal 使用 softmax。

如果不做适配，可能取错维度、用错概率或直接崩溃。

### 问题三：共享配置被临时改写

早期逻辑可能临时修改共享 `opts.curr_step`，给 dataloader、checkpoint 和 teacher 类别数带来隐蔽副作用。

### 审计后的设计升级

主方案从“直接在 Trainer 里写 batch quantile”升级为独立的 `utils/pseudo_label.py`：

- 统一 teacher 输出；
- 明确候选像素；
- 独立解析阈值；
- 独立应用伪标签；
- 明确 `sequential` guard；
- 记录统计；
- 支持离线 artifact 和 teacher hash 校验。

同时提出比 batch quantile 更稳定的正式路线：

> 训练集确定性预扫描 → 类别分位阈值 artifact → 全局回退 → 稀有类收缩 → 绝对上下界 → 可选 margin。

## 3.4 阶段 3：完整工程实现

最终实现了五种策略：

| 策略 | 含义 | 当前研究状态 |
| --- | --- | --- |
| `off` | 不做伪标签 | 必需 baseline |
| `fixed` | 所有旧类共享固定阈值 | 强控制基线，保留 |
| `batch_global` | 当前 batch 所有候选共享分位阈值 | 已验证，降级为消融 |
| `batch_class` | 当前 batch 每个旧类独立分位阈值 | 已验证，淘汰为主方案 |
| `artifact_class` | 预扫描训练集并从 artifact 读取每类阈值 | 工程保留，研究主线降级 |

同时实现：

- `min_conf/max_conf`：阈值绝对上下界；
- `min_pixels`：类别候选太少时回退；
- `shrinkage`：类别阈值向全局阈值收缩；
- `margin_min`：top1-top2 margin 下限；
- teacher checkpoint SHA256；
- artifact 元数据校验；
- per-class candidate/accepted/fallback 统计；
- grid runner；
- calibration → train → summarize 链路；
- 独立 `SUBPATH` 与 `run_manifest.json`。

重要边界：

- `shrinkage`、稀有类回退和正 margin 已实现，但正式实验主要使用 `shrinkage=0`、`min_pixels=1`、`margin_min=0`；
- 它们没有被证明“有效”，也没有必要在 hard-threshold 主假设已失败后继续单独扫参。

## 3.5 阶段 4：Phase A——先补齐 baseline

第一次 `batch_class q0.7` 跑通后，并不能直接说自适应阈值有效，因为当时没有同协议 `off/fixed` 对照。

补齐对照后发现：

- `q=0.7` 只接受约 30% 候选，过于保守；
- `fixed0.6/0.7` 明显更强；
- 降低 q、提高接收率后，batch quantile 才接近 fixed；
- batch 内类别分布、crop 和候选数量导致阈值抖动。

阶段结论：

> 伪标签有用，但 batch 内 quantile 不适合作为正式主方法。

## 3.6 阶段 5：Phase B——offline artifact 与低 q 搜索

为减少 batch 抖动，改用确定性预扫描生成 artifact：

- 统计全训练集候选 confidence；
- 保存 `global_threshold`；
- 保存每个旧类的 `final_threshold`；
- 记录候选数、回退原因、teacher hash 和协议元数据；
- 正式训练只读取 artifact，不在线重新估计。

实验发现 q 越低、接收率越高，结果总体越强：

- `q=0.20`：约接收 80%；
- `q=0.05`：约接收 95%；
- `q=0.005–0.01`：约接收 99%；
- `q=0`：近似全接收。

这时得到一个重要但仍有歧义的线索：

> `artifact_class` 比 fixed0.6 稳定略好，但优势可能来自“整体阈值更低、接收更多”，不一定来自“每类阈值不同”。

## 3.7 阶段 6：多 seed、disjoint 与 15-1

随后补了三类外部验证。

### overlap/disjoint 三 seed

`artifact_class` 相对 fixed0.6 在 overlap 和 disjoint 的六组比较中 6/6 为正，六组平均 `+0.036224 pp`。

这说明“低阈值、高覆盖”不是单个 seed 的偶然现象，但仍不能归因给类别阈值。

### 15-1 overlap 长链

fixed0.6 在 step1-step5 每一步都高于 off，step5 提升达到 `+0.641717 pp`。

这强烈支持伪标签在多步 semantic drift 中的价值，但这里只测了 fixed，不支持 classwise 机制。

## 3.8 阶段 7：matched-global 机制判别

为了消除 Phase B 的归因歧义，为每个 artifact/seed 读取其 `global_threshold`，再用单一全局 fixed 阈值训练。

比较问题变成：

> 在 teacher、seed、总体分位点、模型和 C-RLS 参数一致时，每类不同阈值是否优于一个全局阈值？

六组结果：

- overlap：artifact 极弱领先；
- disjoint：artifact 全部极弱反向；
- 六组平均仅 `+0.002005 pp`；
- 只有预注册 `+0.1 pp` 门槛的约 2%。

这触发 stop-loss：

> 类别特异 hard threshold 不是当前值得继续投入的主要机制。

## 3.9 阶段 8：当前升级方向

当前 C-RLS 只看到 hard class ID。一个伪标签只要被接受，无论 confidence 是 0.51 还是 0.99，都会以完全相同的权重进入：

\[
\min_W \sum_i \lVert\phi_iW-y_i\rVert_2^2+\gamma\lVert W\rVert_2^2.
\]

最新方案改为：

\[
\min_W \sum_i w_i\lVert\phi_iW-y_i\rVert_2^2+\gamma\lVert W\rVert_2^2,
\quad 0\le w_i\le 1.
\]

其中：

- 可见真标签：\(w_i=1\)；
- ignore：\(w_i=0\)；
- 被接受的伪标签：由 confidence 或 confidence×margin 给出连续权重；
- `sample_weight=None`：必须与现有 C-RLS 完全等价。

在实现前先做 raw-mask reliability audit。如果 confidence/margin 与正确率没有稳定关系，就不启动简单加权训练，转向 soft target 或其他路线。

## 6. 哪些方案被淘汰、否定或降级

### 6.1 已淘汰：batch 内 q0.7 classwise

原因：

- 只接收约 30% 候选；
- all mIoU 仅 `0.704535`，明显弱于 fixed0.6 的 `0.707731`；
- batch/crop 类别构成使阈值易抖动；
- 降低 q 后才接近 fixed，说明主要变量是覆盖率。

### 6.2 已降级：batch_global / batch_class

它们仍有消融价值，但不再是正式主方案。

原因：

- 没有稳定超过 fixed0.6；
- offline artifact 更稳定；
- 继续调 batch q 不会回答新的机制问题。

### 6.3 已降级：artifact_class hard threshold

它不是代码垃圾，也不是完全无效：

- 相对 fixed0.6 六组 `6/6` 为正；
- 工程上可复现、可校验；
- 仍可作为历史消融或辅助实现。

但它不再是主攻方法：

- 相对 matched-global 六组平均只有 `+0.002005 pp`；
- disjoint 三组全部轻微反向；
- 效应远低于预注册门槛。

### 6.4 已否定：继续低 q / min_conf / margin 微调

原因：

- q≈0 的区域已经被完整映射；
- 接收率接近 100%，hard selection 几乎不再改变监督集合；
- `min_conf=0.6` 两次都把结果拉回 fixed0.6；
- margin 虽未单独扫完，但 hard-threshold 主机制已失败，继续扫只会增加多重比较。

### 6.5 尚未否定：伪标签本身

相反，现有证据支持保留：

- overlap seed1 artifact 相对 off：`+0.499049 pp`；
- disjoint seed1 artifact 相对 off：`+0.543424 pp`；
- 15-1 step5 fixed0.6 相对 off：`+0.641717 pp`；
- fixed0.6 与高覆盖 artifact 的主要收益都集中在旧类，符合 semantic drift 机制。

### 6.6 尚未验证：可靠性加权与 soft target

这两者都还没有代码或训练结果：

- reliability-weighted C-RLS 是当前首选升级；
- teacher soft target 是前者失败后的二级升级；
- 不能把方案设计写成已经涨点。

## 8. 第一性原理复盘

### 8.1 当前真正有效的变量是什么

实验逐步显示：

```text
接收 30%  → 结果接近 off
接收 75%–90% → 逐步改善
接收 95%–100% → 进入最佳平台
```

因此，当前数据上的主要矛盾是旧类监督缺失，而不是伪标签噪声已经严重到必须强筛选。

### 8.2 hard threshold 为什么触顶

hard threshold 把每个候选压成一个 bit：

```text
通过 = 1
拒绝 = 0
```

当接收率已经接近 100% 时，不同阈值只改变极少像素。即使 teacher confidence 和 margin 还包含大量信息，C-RLS 也完全看不到。

### 8.3 为什么升级要进入 C-RLS

当前被接受的伪标签都以相同权重更新：

\[
X^\top X,\quad X^\top Y.
\]

可靠性加权后：

\[
X^\top D X,\quad X^\top D Y,
\]

其中 \(D=\operatorname{diag}(w_i)\)。

这样可以做到：

- 高可靠伪标签贡献更大；
- 低可靠伪标签仍保留一定召回，但影响更小；
- 不再依赖“阈值小数点是否刚好”；
- 仍保持解析式更新，不引入 SGD。

### 8.4 为什么先做 raw-mask audit

confidence 不一定等于正确率，尤其当前 teacher 使用 BCE sigmoid，各通道并不组成严格概率分布。

如果 confidence/margin 与正确性没有单调关系，直接用它们加权只是在换一种超参数。因此先用训练阶段不可见的完整 raw GT 做只读评价：

- confidence 分箱 precision；
- margin 分箱 precision；
- per-class precision/coverage；
- ECE/reliability curve；
- 错误来自背景、其他旧类还是当前/未来类。

raw GT 只用于诊断，不能进入训练监督或拟合测试专用参数。

## 10. 当前最新方案

### 10.1 核心机制

第一版只比较两个不带额外指数网格的权重：

```text
confidence:
  w_i = top1_confidence_i

confidence_margin:
  w_i = top1_confidence_i × (top1_confidence_i - top2_confidence_i)
```

权重边界：

```text
可见真标签像素 = 1
ignore 像素 = 0
被 matched-global 接受的伪标签 = 上述连续权重
未接受像素 = 保持原标签与原权重
```

W1 中各 candidate 使用与对应 matched-global baseline 完全相同的：

- seed；
- teacher checkpoint；
- fixed threshold；
- 候选定义；
- 接收集合；
- model；
- batch size；
- buffer；
- gamma；
- RHL 设置。

唯一变化是“被接受伪标签进入 C-RLS 时的权重”。

### 10.2 为什么不立刻移除所有阈值

W1 的目标是隔离变量。继续使用已经验证的 seed-specific matched-global threshold，可以让 hard label 集合与 baseline 保持一致，只测试 sample weight。

如果 W1 通过，后续再研究是否能把低阈值进一步降成纯数值安全下界；第一轮不同时改变候选集合和损失权重。

### 10.3 解析更新

令：

\[
X_w=\sqrt{D}X,\qquad Y_w=\sqrt{D}Y.
\]

则：

\[
R_{\text{new}}=(R_{\text{old}}^{-1}+X_w^\top X_w)^{-1},
\]

\[
W_{\text{new}}
=
R_{\text{new}}X_w^\top(Y_w-X_wW_{\text{old}}).
\]

当所有 \(w_i=1\) 时，必须与现有实现数值等价。

## 14. 小白常见问题

### Q1：自适应阈值方法是不是失败了？

“类别特异 hard threshold 作为主创新”没有通过机制门槛；“伪标签研究线”没有失败。

### Q2：为什么 artifact 比 fixed0.6 六组都好，还要停止？

因为 matched-global 证明主要收益来自总体阈值降低和接收更多，而不是每类阈值不同。真实弱收益应保留，但不能误判机制。

### Q3：为什么不再多跑几个 q？

q0、q0.005、q0.01、q0.015 已经把近全接收区域映射清楚。继续细扫不会回答新的问题，只会增加挑最好点的风险。

### Q4：为什么不直接把所有候选都当伪标签？

现有结果说明近全接收是强基线，但低 confidence 像素不一定和高 confidence 像素同等可靠。最新方案保留高覆盖，同时让不同像素以不同权重进入解析目标。

### Q5：raw mask 会不会标签泄漏？

只要 raw mask 只用于离线诊断和评价，不进入训练、不拟合测试专用参数，就不是训练标签泄漏。工具和产物必须明确标注 `audit-only`。

### Q6：为什么先做 confidence/margin，而不是更复杂网络？

它们已经由 teacher 免费提供，接口改动小，能够直接检验“连续可靠性是否比二值筛选更有用”。如果简单机制失败，再升级 soft target，避免一次改太多变量。

### Q7：下一次用户确认后会立刻跑四个训练吗？

不会盲跑。正确顺序是：

```text
实现并运行 W0 audit
  → 审计通过
  → 实现 weighted C-RLS
  → 单元测试 + smoke test
  → dry-run + 资源检查
  → 启动 W1 四个训练
```

任一 gate 失败都先停下分析，不把失败一路带进正式训练。

## 15. 最终阶段性表述

可以严谨地说：

> SegACIL 的自适应伪标签研究已完成从 batch classwise quantile、offline classwise artifact、低分位高覆盖搜索，到 overlap/disjoint 三 seed 与 matched-global 机制判别的闭环。现有证据确认高覆盖伪标签能缓解旧类被背景吞并的 semantic drift，但类别特异 hard threshold 在控制总体阈值后只剩 `+0.0020 pp` 的六组平均差，并在 disjoint 中消失。研究因此停止 hard-threshold 微调，保留伪标签工程栈和 matched-global 基线，转向先审计 teacher reliability，再把连续 confidence/margin 作为 sample weight 引入 C-RLS。

不能说：

- “自适应类别阈值已被三 seed 稳定验证”；
- “artifact 在 overlap 和 disjoint 都优于 global”；
- “只需要继续调 q 就能形成论文创新”；
- “reliability-weighted C-RLS 已经有效”。
