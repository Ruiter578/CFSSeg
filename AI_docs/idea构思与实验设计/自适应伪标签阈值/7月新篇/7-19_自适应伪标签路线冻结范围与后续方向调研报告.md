# 自适应伪标签路线冻结范围与后续方向调研报告

> 日期：2026-07-19
> 项目：SegACIL / CFSSeg
> 分支：`feature/adaptive-pseudo-label`
> 研究模式：快速证据调研与路线决策
> 结论状态：`DECISION — FREEZE THRESHOLD-CENTRIC LINE`
> 证据等级：`CAUTION`（足以做阶段性止损，不等于证明所有未来伪标签方法均无效）

## 1. 执行结论

对用户问题最准确的回答不是简单的“是”或“否”，而是以下四句话：

1. **不是所有伪标签思路都没有效果。** 现有实验支持伪标签作为辅助机制：`15-5 overlap/disjoint seed1` 中 pseudo-on 相对 off 约有 `+0.50/+0.54 pp` 的 all mIoU 方向性收益；`15-1 overlap seed1` 的 fixed 0.6 在五个增量 step 均高于 off，step5 为 `+0.6417 pp`。
2. **迄今所有“在强伪标签基线上继续优化阈值或按可靠性缩放 hard label 影响”的方案，都没有得到达到项目门槛的稳定 all mIoU 收益。** 类别阈值在 matched-global 六组中仅 `+0.0020 pp`；W1 两种连续权重跨 setting 平均仅 `+0.0190 pp` 和 `+0.0339 pp`，均未达到预注册 `+0.1 pp`。
3. **因此应冻结整个“自适应伪标签阈值”方案作为当前研究主线，而不只是冻结某一个 q 或某一种实现。** 不再构思和验证新的 quantile、class-wise threshold、margin threshold、temperature、floor 或相邻阈值变体。
4. **冻结阈值研究不等于删除伪标签。** 高覆盖 pseudo-label 保留为 `overlap/disjoint` 的辅助模块；soft/partial target、背景语义分解或类级影响约束等非阈值想法进入候选池，但当前不实现、不占用 GPU 主队列。主动资源转向更贴合 CFSSeg 核心和 `15-5 sequential` 主协议的 RHL/C-RLS；集成学习应在获得足够不同的强成员后再推进，而不是立刻继续融合现有高度相似的 RHL seed。

一句话决策：

> **主动路线选择“RHL 单模型机制优先，异构集成随后”；对伪标签采取“冻结阈值中心研究、保留辅助模块、暂存非阈值新假设”的三级处理。**

这里必须进一步澄清项目继承关系：**伪标签机制原本就是 CFSSeg 在非
`sequential` 协议下恢复旧类监督的重要模块，本项目提出的是对原有固定阈值的
“自适应阈值改造”，并不是从零新增伪标签。** 因而冻结的是改造路线，不能把
结论误写成“删除或否定原 CFSSeg 伪标签机制”。soft/partial pseudo target
改变的是 target 表达和学习目标，属于更广义的伪监督候选，不属于严格意义上的
自适应阈值方案。

## 3. 到目前为止，究竟哪些东西有效、哪些没有

### 3.1 证据分层

| 层级 | 已验证问题 | 结果 | 当前判断 |
| --- | --- | ---: | --- |
| A. 是否使用伪标签 | pseudo-on vs off | overlap/disjoint seed1 约 `+0.50/+0.54 pp`；15-1 step5 `+0.6417 pp` | 有方向性价值，保留 |
| B. 提高覆盖率是否有用 | low threshold / high coverage vs fixed 0.6 | artifact 相对 fixed 0.6 六组均值 `+0.0362 pp` | 弱正向，但远非核心创新 |
| C. 类别自适应阈值是否有额外价值 | artifact-class vs matched-global | 六组均值 `+0.0020 pp`；disjoint 3/3 轻微反向 | 实际等效，否定为主机制 |
| D. confidence 是否能排序正确性 | W0 raw-mask audit | 两种信号在两 setting 的 Spearman `ρ=1.0`，top-bottom precision 差 `33.9–55.1 pp` | 排序信息真实存在 |
| E. 排序信息能否通过简单样本权重变成 mIoU | W1 weighted C-RLS | `+0.0190/+0.0339 pp`，低于 `+0.1 pp` gate | 路线筛选失败 |

这张表说明一个容易混淆的事实：

> **“伪标签有用”“置信度能判断对错”“继续优化阈值能涨点”是三个不同命题。前两个得到支持，第三个没有。**

### 3.2 W1 不是“完全没动”，而是“动了但净收益不够”

| setting | weighting | Δall | Δold | Δnew |
| --- | --- | ---: | ---: | ---: |
| overlap | confidence | `-0.0059 pp` | `-0.0179 pp` | `+0.0327 pp` |
| overlap | confidence × margin | `-0.0159 pp` | `-0.0471 pp` | `+0.0840 pp` |
| disjoint | confidence | `+0.0439 pp` | `-0.0072 pp` | `+0.2074 pp` |
| disjoint | confidence × margin | `+0.0838 pp` | `-0.0347 pp` | `+0.4629 pp` |

尤其是 disjoint 的 confidence × margin 明显提高了新类，却同时降低了旧类。这证明权重实现确实改变了解析解，也暴露了真正瓶颈：**问题已经从“哪些伪标签正确”转变为“不同伪标签对 old/new 决策边界应该产生多大影响”**。继续改变阈值只是在相同瓶颈外围移动 accepted mask。

### 3.3 第一性原理解释

当前高覆盖基线在 overlap 接受约 99% 候选，在 disjoint 接近 100%。当接受集合几乎饱和后：

\[
\text{继续调阈值} \approx \text{只改变极少量边界像素},
\]

而 C-RLS 仍把 accepted pseudo-label 压成单一 hard class target。W0 证明低 confidence 像素更容易错，但 W1 表明：

\[
\text{correctness ranking} \neq \text{optimal influence ranking}.
\]

低置信像素可能同时是边界、小物体和困难旧类的关键样本。删除或降权它们能减小标签噪声，也可能损失维持旧类决策边界所需的有效样本质量。W1 的 old/new 交换正符合这个解释。

## 4. 外部文献对路线判断的影响

### 4.1 “类别自适应阈值”不是空白点

PLOP 已在持续语义分割中使用旧模型为背景区域恢复旧类，并根据每个旧类的预测熵中位数构造 class-specific threshold（Douillard et al., 2021）<!--ref:douillard2021plop--><!--anchor:section:3.3-->。更早的 MiB 已把增量分割的背景语义漂移明确为核心问题，并通过背景建模和分类器初始化处理该问题（Cermelli et al., 2020）<!--ref:cermelli2020mib--><!--anchor:section:Abstract-->。

因此，即使本项目的 class-wise quantile 最终出现小幅收益，单独依赖“不同类别使用不同阈值”也面临较高的新颖性风险。当前 matched-global 已进一步证明其在 SegACIL 中几乎没有独立贡献。

### 4.2 近年的伪标签研究已从“调一个阈值”转向“如何利用不可靠信息”

U2PL 不直接丢弃不可靠像素，而把它们作为不太可能类别的负样本，并随训练过程调整可靠/不可靠划分（Wang et al., 2022）<!--ref:wang2022u2pl--><!--anchor:section:Abstract-->。ESL 把高熵预测保留为动态 soft label，再用部件级对比约束减少模糊边界（Ma et al., 2023）<!--ref:ma2023esl--><!--anchor:section:Abstract-->。ICCV 2025 的 CSL 则指出固定置信阈值会受过度自信和空间语义断裂影响，并改用样本特异的选择边界与上下文学习（Liu & Liu, 2025）<!--ref:liu2025csl--><!--anchor:section:Abstract-->。

这些工作不直接证明其机制适用于 SegACIL，但共同说明：**继续做 threshold function 的小变体既拥挤，也没有正面对准 W1 暴露的“样本影响与结构信息”问题。**

### 4.3 CISS 主线问题仍是稳定性—可塑性冲突，而非阈值本身

SSUL 通过 unknown class、冻结旧分类器、BCE、pseudo-label 和小型 memory 的组合同时处理背景漂移与稳定性—可塑性（Cha et al., 2021）<!--ref:cha2021ssul--><!--anchor:section:Abstract-->。2025 年 CoMBO 进一步把旧类保持和新类获取的冲突拆到不同分支和目标中（Fang et al., 2025）<!--ref:fang2025combo--><!--anchor:section:Abstract-->。这与 W1 的 old/new 交换现象一致：真正值得建模的是冲突的分解与解析目标中的影响分配，而不是再寻找一个更精细的门槛。

CFSSeg 的主要贡献是冻结表征后的随机高维映射和闭式递归分类头，而非伪标签阈值本身（Li et al., 2025）<!--ref:li2025cfsseg--><!--anchor:section:Abstract-->。因此，对本项目而言，RHL/C-RLS 比阈值更接近方法身份和 `sequential` 主协议。

## 5. 应该冻结哪一层

### 5.1 硬冻结：整个“阈值中心”研究线

以下内容在本阶段一并停止，而不是换一个名字继续：

- fixed / batch-global / batch-class / artifact-class 的新 q 网格；
- per-class quantile、min/max confidence、margin threshold；
- temperature、floor、power、shrinkage 等围绕 accepted mask 或 scalar weight 的小改动；
- 为 W1 补 seed2/3、15-1、DeepLabV3+ 或更多 backbone；
- 根据 disjoint 新类上涨临时改用 new mIoU 作为主 gate。

这意味着：**确认冻结“自适应伪标签阈值”整个方案作为论文级和项目级主动研究线。**

### 5.2 保留：伪标签辅助模块

不删除：

- teacher 从背景中恢复旧类候选的逻辑；
- matched-global 高覆盖强基线；
- `PseudoLabeler`、artifact/hash、stats、manifest、protocol guard；
- weighted C-RLS 的通用实现与测试。

保留理由是 pseudo-on 相对 off 的方向性收益，以及这些工程模块仍可支撑消融、复现和其他目标构造。保留不等于继续扫参。

### 5.3 条件冻结：更广义伪目标研究

下列方向不属于“阈值新 idea”，可以保留为 future work，但当前不启动：

1. **闭式 partial/soft target**：让 teacher 的候选类别集合或概率分布进入 C-RLS target，而不是 hard class ID。
2. **类级有效样本质量约束**：直接约束 old/new 或 per-class 对 \(X^\top X\) 与 \(X^\top Y\) 的贡献，解决 W1 暴露的影响失衡。
3. **区域/实例一致性目标**：用连通区域、边界或 prototype agreement 描述结构可靠性，而不是仅看像素 confidence。

其中第 2 项更适合并入 C-RLS/RHL 主线；第 1、3 项已有大量相邻研究，只有形成“闭式充分统计量下的新目标定义”才可能成为本项目自己的研究问题。

### 5.4 分支与主线处理边界

- `feature/adaptive-pseudo-label` 作为完整实验、实现和失败证据的归档分支保留，
  不整体合并到 `main`。
- `main` 只接收已经确认且无争议的阶段结论、证据索引和路线状态文档。
- `PseudoLabeler`、weighted C-RLS、protocol guard、teacher output adapter 等
  虽有工程价值，但都会改变或重构训练关键路径；在没有单独确认前不移植到
  `main`，也不以“默认关闭”为理由绕过审查。
- RHL 后续从最新 `main` 创建独立 feature 分支；RHL 代码只在该分支内开发和
  验证，达到预注册 gate 并完成独立审查前不合并主线。

这一处理把“保留研究资产”和“批准进入主线”明确分开：前者已经确定，后者需要
逐项证据和确认。

## 6. 三个选项的机会成本比较

| 选项 | 主协议匹配 | 已有证据空间 | 新颖性空间 | 工程/实验成本 | 建议 |
| --- | --- | --- | --- | --- | --- |
| A. 继续围绕阈值构思其他 idea | 低：不适用于 sequential | 低：接受率饱和且两轮机制 gate 失败 | 低：class-wise/adaptive selection 已拥挤 | 表面低，长期试错高 | 拒绝 |
| B. 立即转做 soft/partial pseudo target | 低到中：仍主要服务 overlap/disjoint | 中：尚未在 SegACIL 验证 | 中：必须突出 closed-form 适配 | 高：背景/BCE/target 语义复杂 | 候选池，不立即执行 |
| C. 转向 RHL/C-RLS，再构建异构集成 | 高：直接服务 sequential | 中：BOA 有弱单点信号，PGH 等结构假设未充分验证 | 中到高：贴合 CFSSeg 方法身份 | 中到高，但可严格分阶段止损 | 当前主线 |

这里的“选择 C”不表示 RHL 已经被证明成功。现有 RHL-SE 静态融合相对强 batch32 单模型没有优势，BOA orthogonal 也只有单次 `+0.0789 pp` 的弱信号。选择 C 的理由是：

- 阈值路线的有效信息增益已经接近饱和；
- RHL/C-RLS 直接作用于项目的解析学习核心；
- 它覆盖主目标 `sequential`；
- BOA、prototype-guided feature 和 class-aware sufficient statistics 是机制层变化，不是重复扫同一参数；
- 后续可以用同样严格的 gate 快速淘汰无效分支。

## 7. 后续行动路线

### 7.1 立即执行的研究顺序

#### 第一优先级：RHL 单模型机制

先在 `feature/rhl-se-boa-p0-p1` 上做 BOA-RHL 1.5 的 paired-seed 复核：

- legacy baseline vs orthogonal/legacy；
- `rhl_seed=1/2/3`，global seed、step0、batch、BUFFER、GAMMA 全部固定；
- 主判断使用同 seed paired delta；
- 建议预注册：平均 Δall 至少 `+0.1 pp`、至少 2/3 为正、任一 seed 不低于 `-0.05 pp`；
- 不通过即停止 orthogonal 的 scale/gamma/容量扩展，不用新网格“救结果”。

随后只保留一个独立、机制不同的单模型候选，例如 PGH-RHL-lite 的 K=1 cosine + 公平容量对照。不要同时铺开 BOA、PGH、PowerNorm、class weight 的组合网格。

#### 第二优先级：C-RLS 目标层

W1 的 disjoint 结果提示 old/new 影响分配可能比正确性权重更关键。若要研究 class-aware C-RLS，应作为独立的解析目标方法，在 sequential 真标签协议先验证；不要继续挂在“伪标签阈值”名下。

#### 第三优先级：异构集成

当前 RHL seed 成员全局 disagreement 不足 1%，已有静态融合没有超过强 batch32 单模型。因此不建议现在继续做更多 seed、全局权重或 class-wise 权重。

只有获得以下任一条件后再恢复集成：

- 两个以上通过单模型 gate、机制不同的成员；
- Snapshot backbone 带来明显更高的成员差异；
- validation 上能预注册并证明 routing 能稳定捕获 oracle gap。

集成的顺序应为：强成员与多样性审计 → uniform baseline → 固定的 val-driven gate → test 一次性评估。

### 7.2 伪目标路线的重启门槛

只有同时满足以下条件，才允许重新开启伪标签相关研究：

1. 研究问题不再是“如何设阈值”；
2. 一页 first-principles 设计明确解释 W1 的 old/new trade-off；
3. 数学上定义 target 或 sufficient-statistics contribution，说明与 PLOP/U2PL/ESL/CSL 的差异；
4. 先做零训练的可证伪诊断，再做 overlap/disjoint seed1 的一次 S0；
5. S0 跨 setting 平均 Δall 至少 `+0.1 pp`，任一 setting 不低于 `-0.05 pp`；
6. S0 不通过即停止，不补 seed、不扫小参数。

在 RHL 主线完成至少一个独立机制筛选前，不触发这条重启流程。

## 8. 反方审查与结论强度

### 反方意见一：W1 只有 seed1，冻结是否过早？

单 seed 不足以证明普遍无效，但 W1 本来就是预注册的低成本 screening。两种方法都只达到目标门槛的 19%–34%，overlap 方向为负，且已经出现 old/new 交换。继续补 seed主要减少不确定性，不能修复假设与目标之间的错位。对资源分配而言，冻结是合理的；对科学表述而言，只能写“停止当前路线”，不能写“所有 weighted pseudo-label 永远无效”。

### 反方意见二：disjoint 新类 `+0.4629 pp` 难道不值得追？

这个信号值得保留为机制诊断，但项目预注册主指标是 all mIoU，而且旧类下降、overlap 不受益。事后改主指标会产生选择性汇报。更合理的用法是把它转化为“old/new influence conflict”的新 C-RLS 假设，而不是继续调伪标签权重。

### 反方意见三：RHL 目前也没有明显涨点，为什么转向它？

RHL 不是已确认赢家，只是当前机会成本更优的研究对象。它适用于主协议、位于 CFSSeg 核心、仍有未被饱和验证的结构假设；因此应获得一个同样有 stop gate 的短周期，而不是无限投入。

### 反方意见四：soft target 可能正好有效，为什么不马上做？

soft target 与 hard weight 确实不同，但背景、新类真值、BCE sigmoid 非归一化输出和 teacher 旧类分布如何合并尚无唯一答案。外部文献也已有大量 soft/partial-label 设计。没有先形成闭式目标下的独立假设就实现，极易重新进入自由度过多的小改动循环。

---

## 9. 最终决策清单

| 对象 | 决策 | 状态 |
| --- | --- | --- |
| adaptive class-wise hard threshold | 冻结，不再验证 | `STOP` |
| confidence/margin scalar weighted hard target | 冻结，不进 W2 | `STOP` |
| 整个“自适应伪标签阈值”论文主线 | 冻结 | `FROZEN` |
| pseudo-label 辅助模块 | 保留，不主动优化 | `MAINTAIN` |
| 自适应分支中的训练关键代码 | 分支归档，暂不移植 main | `ARCHIVE / DEFER` |
| soft/partial/structured pseudo target | 候选池，满足重启 gate 才讨论 | `PARKED` |
| RHL/C-RLS 单模型机制 | 下一主动主线，逐方案严格止损 | `ACTIVE NEXT` |
| 现有同质 RHL seed 集成 | 暂缓 | `WAIT FOR DIVERSITY` |
| 异构/快照/机制组合后的集成 | 强成员和多样性通过后再做 | `CONDITIONAL` |

最终回答：

> **确认冻结的范围应是“自适应伪标签阈值这一整个研究方案”，而不是只冻结当前 q；但不冻结伪标签作为 overlap/disjoint 辅助机制，也不在理论上永久排除所有非阈值伪目标研究。下一步不再继续围绕阈值构思其他 idea，主动转向 RHL/C-RLS 单模型机制；集成学习等待强且有差异的成员后再启动。**

## 10. 局限性

- W1 是 seed1 screening，适合止损，不是跨数据集普遍性证明。
- `15-1` 与部分历史 artifact 是方向性证据，尚非最终 clean-commit 论文矩阵。
- 外部文献中的 SSL、SGD 和 mask-classification 方法不能直接移植到 closed-form C-RLS。
- 文献调研用于判断研究拥挤度和机制趋势，没有执行系统综述或 meta-analysis。
- RHL 后续路线本身仍需预注册 gate；本报告不承诺其必然涨点。
