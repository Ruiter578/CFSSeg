# 自适应伪标签阈值路线再审与机制判别实验方案

## 1. 文档目的

本文对 SegACIL 当前自适应伪标签阈值路线进行一次面向决策的复核，回答以下问题：

1. 当前方法已经验证到什么程度，哪些结论可信，哪些结论仍缺少严格对照；
2. 现有实验是否足以证明“类别自适应阈值”本身有效；
3. 是否应继续细化分位数、阈值下界等局部超参数；
4. 何时保留现有方法，何时升级底层设计，何时终止该路线；
5. 下一轮实验如何用最小成本给出可证伪的机制判断。

本轮研究不预设自适应阈值必须成为论文级核心创新。判断标准只有一个：在协议一致、控制变量充分、结果可复现的条件下，它是否带来稳定且有实际意义的提升。真实但较小的正向收益可以支持保留该模块；多次严谨验证后仍无明显收益，则必须停止重复扫参，转向根因分析、机制升级或替代路线。

## 2. 当前实现与实验状态

### 2.1 当前方法

当前伪标签模块已经形成从阈值生成到训练消费、再到结果汇总的完整链路，主要包含：

- `off`：关闭伪标签；
- `fixed`：所有旧类使用统一固定置信度阈值；
- `batch_class`：在训练批次内按类别估计阈值；
- `artifact_class`：训练前离线统计每个旧类的置信度分布，将类别阈值及其来源固化为 artifact，训练阶段只读取该 artifact；
- `artifact_class` 的边界控制参数：`quantile`、`min_conf`、`max_conf`、`min_pixels`、`shrinkage`、`margin_min`。

其中 `artifact_class` 相比在线估计更适合作为研究对象，因为阈值生成与训练解耦，能够记录 teacher checkpoint、类别阈值、全局阈值、候选像素数和回退来源，具备复现实验与审查机制的基础。

### 2.2 已完成实验规模

当前仓库中可核验的伪标签实验目录共 43 个：

- VOC 15-5 overlap：29 个；
- VOC 15-5 disjoint：11 个；
- VOC 15-1 overlap：3 个。

另有 20 个阈值 artifact。判断实验状态时应以 `test_results_*.json`、`pseudo_label_stats.json` 和 `run_manifest.json` 为准，不以可能滞后的阶段性文字说明为准。

截至本次复核，没有新的自适应伪标签实验正在运行，也没有晚于上一轮汇总的新结果。

### 2.3 已有正向证据

在 VOC 15-1 overlap 的 step5 上，已有单次结果显示：

| 方法 | all mIoU | 相对 off |
| --- | ---: | ---: |
| pseudo off | 0.697703 | — |
| fixed 0.6 | 0.704120 | +0.006417，即 +0.6417 pp |

这说明伪标签机制本身在更长增量链路上可能具有实际价值。该证据支持继续研究伪标签，但不能直接证明类别自适应阈值优于合适的固定阈值，且现有 15-1 结果来自旧的非干净工作树运行，只能视为方向性证据，不能作为最终论文结果。

### 2.4 `artifact_class` 相对 fixed 0.6 的现有差异

15-5 的三 seed 配对结果如下：

| setting | seed | fixed 0.6 | artifact | artifact - fixed 0.6 |
| --- | ---: | ---: | ---: | ---: |
| overlap | 1 | 0.707731 | 0.708082 | +0.000351 |
| overlap | 2 | 0.707608 | 0.708017 | +0.000409 |
| overlap | 3 | 0.703632 | 0.704058 | +0.000426 |
| disjoint | 1 | 0.694639 | 0.694872 | +0.000233 |
| disjoint | 2 | 0.693842 | 0.694181 | +0.000339 |
| disjoint | 3 | 0.692441 | 0.692858 | +0.000417 |

由此得到：

- overlap 三 seed 均值增益约为 `+0.000395`，即 `+0.0395 pp`；
- disjoint 三 seed 均值增益约为 `+0.000330`，即 `+0.0330 pp`；
- 六组结果方向一致，但绝对幅度很小；
- overlap `q=0.01` 的接收率约为 98.94%–99.10%；
- disjoint `q=0.00` 的接收率约为 99.99%–100%。

这是一组值得保留的弱正向信号，但还不是对“类别自适应”机制的充分证明。因为 artifact 方法不仅使用类别间不同的阈值，同时也把总体阈值水平降到了远低于 0.6 的区间。当前实验混合了两个因素：

1. 更低的总体阈值带来的高接收率；
2. 类别间阈值差异带来的自适应选择。

如果不补充与 artifact 同总体分位点的全局固定阈值对照，就无法判断现有增益主要来自哪一个因素。

## 3. 第一性原理与对抗性审查

### 3.1 方法真正改变了什么

对候选旧类像素 \(i\)，当前硬阈值方法可写为：

\[
m_i = \mathbb{1}[p_i \ge \tau_{c_i}]
\]

其中 \(p_i\) 是 teacher 对预测类别 \(c_i\) 的置信度，\(\tau_{c_i}\) 是固定阈值或类别阈值。方法对训练链路的直接作用不是“提升 teacher 质量”，而是决定哪些候选像素进入伪标签集合。

因此，类别自适应阈值成立至少需要以下机制链条：

1. 不同旧类的置信度分布或校准误差确实不同；
2. 这种差异对应不同的伪标签精度—覆盖率权衡，而不只是数值尺度不同；
3. 类别阈值选择出的额外像素对后续学习有净正贡献；
4. 该贡献大于随机种子波动、总体接收率变化及优化噪声。

已有 artifact 已证明第 1 点的“置信度分布不同”，但尚未直接证明第 2–4 点。

### 3.2 为什么继续细扫低分位数价值很低

当前最优区域已经逼近“接收全部候选像素”的边界。此时继续测试 `q=0.005`、`0.010`、`0.015` 等相邻分位点，主要改变的是不到几个百分点的候选像素，已有结果差异只有万分位量级。继续微调会产生三个问题：

- 多重比较增大偶然最优的概率；
- 实验回答的是“哪个小数点更好”，而不是“类别自适应为何有效”；
- 即使再找到一个略高点，也无法排除总体低阈值或随机波动的解释。

因此，低分位数精细扫描在当前阶段应停止。只有当机制判别实验确认类别差异确实提供稳定增益后，才有理由在正式协议上进行有限调参。

### 3.3 当前方案的主要风险

#### 风险一：收益可能仅来自更低的全局阈值

固定 0.6 与 artifact 的平均阈值、接收率均不匹配。现有 `artifact - fixed 0.6` 差异不能归因于类别自适应。

#### 风险二：接收率过高使“筛选”退化

当接收率接近 100% 时，阈值模块几乎不再承担排除噪声的职责。此时微弱变化可能来自极少数边界像素，而不是稳健的可靠性建模。

#### 风险三：置信度不等于正确率

按预测置信度分位点设阈值，只保证不同类别保留相似比例或覆盖某一尾部，不保证不同类别达到相似伪标签精度。若 teacher 存在类别相关失校准，简单分位数可能放大而非修复噪声。

#### 风险四：现有主协议边界不支持泛化结论

当前伪标签路径针对 overlap/disjoint；VOC 15-5 sequential 代码路径会显式跳过该机制。因而现阶段不能将伪标签结论外推到项目的 sequential 主线，更不能让不执行的路径静默充当验证。

#### 风险五：旧运行谱系不够干净

当前历史 artifact 和部分 15-1 结果来自非干净工作树。它们可以用于机制筛选和路线决策，但通过筛选后仍需在最新主线、固定提交和可追溯 manifest 上做正式重放。

## 4. 候选路线与决策

### 路线 A：继续围绕 `quantile/min_conf` 做小范围扫描

优点是改动最小，能够复用已有 runner。缺点是当前低分位区域已经接近全接收，新增实验难以提供机制信息，且容易形成低价值重复验证。

**结论：终止。** 在类别自适应机制得到严格支持前，不再增加低分位数微调实验。

### 路线 B：先做同总体分位点的全局固定阈值配对对照

从每个 seed 的 artifact 中读取 `global_threshold`，用该值替换类别阈值，其他设置完全一致。这样类别自适应组和全局固定组来自同一 teacher、同一数据遍历 seed 和同一总体分位点，主要差别只剩“是否按类别分配不同阈值”。

优点是成本低、无需修改模型代码、能够直接证伪当前机制解释。缺点是仍属于硬阈值研究，不能解决置信度失校准和二值接收的信息损失。

**结论：立即执行，作为下一轮唯一实验。**

### 路线 C：升级为可靠性加权的 C-RLS

若路线 B 表明类别阈值没有稳定价值，则不再继续阈值微调，而把研究对象从“是否接收一个像素”升级为“该像素以多大可信度进入解析学习的充分统计量”。

设特征为 \(\phi_i\)，伪标签为 \(y_i\)，可靠性权重为 \(w_i \in [0,1]\)，加权闭式目标为：

\[
\min_W \sum_i w_i \lVert \phi_i W-y_i \rVert_2^2+\gamma\lVert W\rVert_2^2
\]

对应充分统计量：

\[
A=\sum_i w_i\phi_i\phi_i^\top+\gamma I,\qquad
B=\sum_i w_i\phi_i y_i^\top,\qquad
W=A^{-1}B
\]

该设计保留连续可靠性信息，避免硬阈值在边界处发生不连续决策，也能把伪标签可信度直接作用到 C-RLS 的训练目标，而不只是作用于数据预处理。

**结论：作为 stop-loss 后的底层升级路线。** 路线 B 未达到预设门槛时进入该路线，不能一边判定硬阈值收益不足，一边继续做同类微调。

### 5.6 主评价量

对每个 seed 定义：

\[
\Delta_s =
\operatorname{mIoU}(\text{artifact\_class}_s)
-\operatorname{mIoU}(\text{matched-global-fixed}_s)
\]

分别报告 overlap、disjoint 及六组总体的：

- 每 seed `all mIoU`、`old mIoU`、`new mIoU`；
- \(\Delta_s\)；
- 均值、标准差；
- artifact 与 fixed 的接收率差异；
- 六组中 artifact 胜出的组数；
- 结果来源 JSON 和 teacher SHA256。

禁止只汇报最优 seed，禁止把未完成运行纳入均值，禁止用旧 summary 替代实际结果文件。

### 5.7 预注册决策门槛

考虑现有 `artifact - fixed 0.6` 只有约 `0.03–0.04 pp`，本轮采用以下 stop-loss 规则：

#### 判定 A：保留类别自适应并继续正式验证

同时满足：

- overlap 和 disjoint 两个 setting 中，artifact 的三 seed 平均值均高于 matched-global fixed；
- 六组总体平均 \(\overline{\Delta}\ge 0.001\)，即至少 `+0.1 pp`；
- 不出现某一 setting 明显反向、仅由单个 seed 拉高均值的情况。

后续动作：

1. 将 `artifact_class` 保留为独立候选模块；
2. 在 15-1 上补充 artifact 的干净运行；
3. 将实现整合到最新主线后进行正式 clean replay；
4. 再研究校准、类别可靠性等机制，而不是继续做 q 的微小扫描。

#### 判定 B：存在一致弱正向，但低于 `+0.1 pp`

表现为多数或全部 seed 方向为正，但 \(\overline{\Delta}<0.001\)。

后续动作：

1. 将类别阈值降级为辅助工程选项，不作为主要创新主张；
2. 保留现有代码和证据，不删除真实正向结果；
3. 停止 hard-threshold 超参数实验；
4. 转入可靠性加权 C-RLS 的机制设计与最小实现。

#### 判定 C：持平、反向或 setting 间结论冲突

后续动作：

1. 不再将类别分位阈值视为核心有效机制；
2. 先核查候选像素集合、接收率、类别分布、teacher SHA 和结果来源；
3. 若工程一致性无误，则判定当前 hard classwise threshold 路线失败；
4. 基于失败原因选择可靠性加权 C-RLS 或其他机制，不再追加 q/min_conf 微调。

setting 间方向相反时，不允许立刻取平均掩盖冲突。必须先分析 overlap 与 disjoint 的标签可见性、候选像素构成及 teacher 置信度分布差异。

## 6. 若硬阈值路线失败，底层设计如何升级

### 6.1 升级目标

底层升级不是为了增加方法复杂度，而是针对当前失败机制：

- 硬阈值丢失连续置信度信息；
- 类别分位点不等价于伪标签正确率；
- 接近全接收时，阈值本身失去筛选作用；
- 当前阈值只改变样本是否进入训练，未进入解析学习的目标权重。

### 6.2 最小可行设计

第一版可靠性加权 C-RLS 只引入一个明确接口：

\[
(\phi_i, y_i) \longrightarrow (\phi_i, y_i, w_i)
\]

其中：

- 真值像素或可信旧类标签使用 `w=1`；
- 伪标签像素使用由 teacher 置信度映射得到的 `w∈[0,1]`；
- ignore 像素使用 `w=0`；
- 权重进入 C-RLS 的 \(A\) 和 \(B\) 累积，不只用于 mask。

首轮只比较三组：

1. pseudo off；
2. 当前最佳 hard pseudo-label；
3. reliability-weighted pseudo-label。

权重函数应先采用单调、参数少、可审计的形式，例如经温度校准后的置信度或分段线性权重，不同时引入类别先验、动态温度和复杂元学习。

### 6.3 升级前必须回答的问题

1. 当前 RLS sufficient statistics 在代码中的唯一写入点在哪里；
2. 像素级权重是否会在池化、采样或特征压缩阶段被丢失；
3. 权重是作用于样本、像素、类别原型还是解析目标；
4. checkpoint 与 run manifest 如何保存权重策略和校准参数；
5. eval/infer 是否需要恢复这些参数；
6. 旧 checkpoint 在未提供权重配置时如何保持 `w=1` 的兼容行为。

未回答这些问题前，不应直接堆叠新的可靠性模块。

## 9. 当前结论

1. 伪标签作为整体机制已有值得继续研究的正向信号，尤其是 15-1 step5 的 fixed 0.6 结果；
2. `artifact_class` 在 15-5 两种 setting、三 seed 上均略优于 fixed 0.6，但增益只有约 `0.03–0.04 pp`，尚不足以证明类别自适应本身有效；
3. 当前实验缺少 matched-global fixed 这一关键机制对照，继续细扫 q 的信息价值很低；
4. 下一轮只新增 6 个 seed-specific matched-global fixed 实验；
5. 若类别自适应相对 matched-global fixed 的总体平均增益达到 `+0.1 pp` 且两个 setting 一致，则保留并做正式验证；若仅有不足 `+0.1 pp` 的弱正向，则作为辅助模块保留并转向可靠性加权 C-RLS；若持平或反向，则终止 hard classwise threshold 核心路线；
6. 当前底层代码不需要在机制对照前立即升级，但已经达到必须设置升级触发条件的阶段。六组对照给出否定或弱收益结论后，应果断把硬筛选升级为进入 C-RLS 充分统计量的连续可靠性权重。
