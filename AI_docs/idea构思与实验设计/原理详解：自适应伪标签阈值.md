问题：
没看懂你之前说的自适应伪标签阈值的原理，浅显易懂地解释一下

## 核心一句话

**自适应伪标签阈值，就是不要死板地规定“旧模型置信度超过 0.7 才相信”，而是根据当前 batch / 当前类别的置信度分布，动态决定“哪些预测相对最可靠”。**

可以把它理解成：

```text
固定阈值：所有考试都要求 70 分及格。
自适应阈值：每次考试根据整体难度，只取每个班前 30% 的答案。
```

---

# 1. 先搞清楚伪标签在干什么

在类增量分割里，step1、step2 之后有一个麻烦：

```text
当前 step 的标注只标新类；
图像里出现的旧类，经常被标成 background。
```

举个例子。

step0 学过：

```text
person, dog, car, chair ...
```

step1 要学新类：

```text
bus
```

但是 step1 的训练图像里可能同时有 `person` 和 `bus`。由于当前 step 只监督 `bus`，所以标签可能是：

```text
bus 像素：bus
person 像素：background
真正背景：background
```

这就有问题了：

```text
person 明明是旧类，却在新 step 里被当成 background。
```

如果直接拿这个标签训练，模型会越来越倾向于把旧类当背景，这就是 semantic drift（语义漂移）。

所以 CFSSeg 会用旧模型帮忙：

```text
当前标签说：这个像素是 background
旧模型说：不，它很像 person，而且我很确定
那就把这个 background 改成 person 伪标签
```

伪标签的核心逻辑就是：

```text
如果当前标签是 background，
并且旧模型高置信预测它是旧类，
那就把它从 background 改回旧类。
```

---

# 2. 固定阈值是什么意思？

假设旧模型对某个 background 像素的预测是：

```text
person: 0.82
dog:    0.10
car:    0.05
```

如果固定阈值是：

```text
threshold = 0.7
```

因为 `0.82 > 0.7`，所以我们相信旧模型，把这个像素改成：

```text
person
```

再看另一个像素：

```text
chair: 0.65
```

因为 `0.65 < 0.7`，所以不相信它，继续保持：

```text
background
```

这就是固定阈值：

```python
if old_model_confidence >= 0.7:
    use_pseudo_label
else:
    keep_original_label
```

它简单，但问题也明显。

# 3. 为什么固定 0.7 不一定合理？

因为不同情况的置信度分布不一样。

## 情况 A：这个 batch 很难

旧模型整体都不太自信：

```text
person 像素最高置信度：0.62
dog 像素最高置信度：0.58
chair 像素最高置信度：0.55
```

如果你死守 `0.7`：

```text
没有任何像素会被改成旧类
```

结果就是：

```text
旧类仍然被当成 background
```

这会伤害旧类保持能力。

---

## 情况 B：这个 batch 很容易

旧模型特别自信：

```text
大量像素置信度都在 0.9 以上
```

这时 `0.7` 可能太宽松，会把很多其实不可靠的边界像素也收进来。

结果是：

```text
错误伪标签变多
```

对于 CFSSeg 尤其危险，因为增量阶段是闭式解更新，伪标签错误会直接进入矩阵求解，不像 SGD 那样多轮迭代中可能被部分抵消。

---

## 情况 C：不同类别难度不同

旧模型对 `person` 很自信：

```text
person: 常见置信度 0.85 ~ 0.95
```

但对 `chair` 不自信：

```text
chair: 常见置信度 0.55 ~ 0.70
```

如果统一用 `0.7`：

```text
person 很容易被伪标签恢复
chair 几乎恢复不了
```

这样会导致不同旧类之间不公平。

---

# 4. 自适应阈值的直觉

自适应阈值不问：

```text
这个像素有没有超过固定 0.7？
```

而是问：

```text
在当前这批候选像素里，它是不是属于相对最可靠的一部分？
```

比如当前 batch 里，旧模型把一些 background 像素预测成 `person`，置信度如下：

```text
0.91, 0.88, 0.84, 0.80, 0.76, 0.70, 0.62, 0.55, 0.49, 0.41
```

如果设置：

```text
只保留最高的 30%
```

那么只拿：

```text
0.91, 0.88, 0.84
```

这些像素作为 person 伪标签。

这时阈值不是手写的 `0.7`，而是从这组数据里自动算出来的，大概是：

```text
threshold ≈ 0.84
```

所以它是动态的。

# 5. 用“分位数”解释最简单

我之前提到的 `quantile`，中文叫 **分位数**。

假设你设置：

```text
pseudo_label_quantile = 0.7
```

意思是：

```text
把候选像素按置信度从低到高排序，取 70% 位置上的置信度当阈值。
```

这样会保留大约最高的 30%。

举例：

```text
置信度排序：
0.41, 0.49, 0.55, 0.62, 0.70, 0.76, 0.80, 0.84, 0.88, 0.91
```

70% 位置大约是：

```text
0.80 或 0.84
```

于是：

```text
大于这个值的像素保留为伪标签
低于这个值的像素丢弃
```

所以：

```text
quantile = 0.7 约等于保留 top 30%
quantile = 0.8 约等于保留 top 20%，更严格
quantile = 0.6 约等于保留 top 40%，更激进
```

# 6. 全局自适应 vs 按类别自适应

有两种常见做法。

## 6.1 全局自适应阈值

把所有旧类候选像素混在一起，算一个阈值。

例如：

```text
person 候选像素
dog 候选像素
chair 候选像素
car 候选像素
```

全部放一起排序，取 top 30%。

优点：

```text
简单
稳定
实现容易
```

缺点：

```text
强势类别会挤压弱势类别
```

比如 person 置信度普遍高，chair 置信度普遍低，那么最后伪标签可能几乎都是 person，没有 chair。

---

## 6.2 按类别自适应阈值

对每个旧类单独算阈值。

例如：

```text
person 单独取 top 30%
dog 单独取 top 30%
chair 单独取 top 30%
car 单独取 top 30%
```

这样更公平。

举例：

```text
person 候选置信度：
0.95, 0.91, 0.88, 0.82, 0.79
person 阈值可能是 0.88

chair 候选置信度：
0.72, 0.68, 0.63, 0.59, 0.54
chair 阈值可能是 0.63
```

你会发现：

```text
person 用高阈值
chair 用低阈值
```

这不是放宽标准乱收，而是因为 chair 本来就比 person 难，旧模型对它整体没那么自信。按类别自适应可以让 chair 至少有一部分相对可靠像素被恢复出来。

# 7. 为什么还需要 min_conf 和 max_conf？

如果只用分位数，会有两个极端问题。

## 极端 1：整体都很烂

假设某个类别的候选置信度是：

```text
0.21, 0.25, 0.28, 0.30, 0.33
```

即使取 top 30%，也可能保留：

```text
0.30, 0.33
```

但这些其实都不可靠。

所以要设置最低阈值：

```text
min_conf = 0.5
```

意思是：

```text
再怎么自适应，低于 0.5 的也不要。
```

---

## 极端 2：整体都很高

假设候选置信度是：

```text
0.96, 0.97, 0.98, 0.99
```

分位数阈值可能非常高，导致保留像素太少。

所以可以设置最高阈值：

```text
max_conf = 0.95
```

意思是：

```text
阈值最高不要超过 0.95，避免过度保守。
```

最终阈值可以理解成：

```text
先按分位数算一个动态阈值
再把它限制在 [min_conf, max_conf] 之间
```

例如：

```text
算出来 0.42，但 min_conf=0.5，所以最终用 0.5
算出来 0.98，但 max_conf=0.95，所以最终用 0.95
算出来 0.83，就直接用 0.83
```

# 8. 整个逻辑用伪代码表示

固定阈值：

```python
for pixel in background_pixels:
    if old_model_score[pixel] >= 0.7:
        label[pixel] = old_model_pred_class[pixel]
```

自适应阈值：

```python
for each old_class:
    candidates = background pixels predicted as this old_class

    scores = old_model_confidence[candidates]

    threshold = quantile(scores, 0.7)   # 例如保留 top 30%
    threshold = clamp(threshold, min=0.5, max=0.95)

    for pixel in candidates:
        if old_model_score[pixel] >= threshold:
            label[pixel] = old_class
```

它的关键差别是：

```text
固定阈值：所有类别、所有 batch 都用同一个 0.7
自适应阈值：每个 batch / 每个类别自己算一个合适阈值
```

# 9. 用一个完整例子看

假设当前 step 要学新类 `bus`。

原始标签：

```text
bus: 正确标注为 bus
person: 被错误标成 background
chair: 被错误标成 background
真正背景: background
```

旧模型预测 background 区域时得到：

```text
person 候选像素置信度：
0.92, 0.89, 0.86, 0.61, 0.52

chair 候选像素置信度：
0.69, 0.65, 0.61, 0.48, 0.43
```

如果固定阈值是 0.7：

```text
person: 0.92, 0.89, 0.86 被恢复
chair: 一个都恢复不了
```

结果：

```text
person 得救了
chair 继续被当 background
```

如果按类别自适应，保留每类 top 40%，并设置 `min_conf=0.5`：

```text
person 阈值大约 0.86
保留：0.92, 0.89, 0.86

chair 阈值大约 0.61
保留：0.69, 0.65, 0.61
```

结果：

```text
person 被恢复
chair 也被恢复一部分
```

这就是自适应阈值的意义。

# 10. 它到底想解决什么？

自适应伪标签阈值想平衡两个东西：

## 目标 1：召回旧类

不要让旧类像素继续被当成 background。

如果阈值太高：

```text
伪标签太少
旧类恢复不回来
```

## 目标 2：控制噪声

不要把错误预测也当成真标签。

如果阈值太低：

```text
伪标签太多
错误标签污染闭式解头
```

自适应阈值就是在做这个折中：

```text
每次只选当前最可靠的一部分旧类预测
```

# 11. 对 CFSSeg 为什么有用？

CFSSeg 的增量阶段不是普通 SGD 微调，而是：

```text
特征 -> RHL -> 闭式解分类头
```

闭式解会直接根据标签矩阵 (Y) 求解分类权重。

如果伪标签太差，相当于给闭式解喂了错误监督：

```text
错误 pseudo-label -> 错误 E^T Y -> 错误分类头权重
```

所以 CFSSeg 里伪标签质量很关键。

固定阈值的问题是：

```text
简单但僵硬
```

自适应阈值的优点是：

```text
根据当前数据难度自动调节伪标签数量和质量
```

它不改变 CFSSeg 的主公式，只改变：

```text
哪些 background 像素可以被改成旧类伪标签
```

所以属于比较稳的“小改动”。

# 12. 最后用一句话记住

固定阈值像这样：

```text
超过 0.7 我才信你。
```

自适应阈值像这样：

```text
我不固定看 0.7，而是看你在当前这一批、当前这个类别里是不是最可靠的那一部分。
```

这就是它的全部直觉。
