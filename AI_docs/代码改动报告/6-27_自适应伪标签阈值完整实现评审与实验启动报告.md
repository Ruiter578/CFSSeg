## 1. 执行结论

本次已完成自适应伪标签阈值的完整代码实现、单元测试、脚本接入、CodeRabbit review 和正式 tmux 实验启动。

关键结论：

- 当前可以先把自适应伪标签作为独立支线推进；它不应默认并入 `15-5 sequential` 主线。
- `sequential + pseudo_label on` 已做显式拒绝，避免“看似启用、实际不合理或 no-op”的实验。
- 首个正式实验选择 `15-5 overlap + batch_class(q=0.7)`，因为当前本机没有可复用的 overlap/disjoint step0 checkpoint，所以 runner 会先跑 overlap step0，再跑 step1 伪标签实验。
- 代码默认 `USE_PSEUDO_LABEL=0`，原 sequential/RHL/V3/V3+ 路径不受影响。

## 2. 主要代码改动

### 2.1 伪标签核心模块

新增：

```text
utils/pseudo_label.py
```

实现内容：

- `extract_teacher_probabilities()`：统一处理 DeepLab `(logits, feature_dict)` 与 AIR `NHWC` 输出，并按 `bce_loss/ce_loss/focal_loss` 转概率。
- `compute_pseudo_label_candidates()`：只允许 `labels == 0` 且 teacher top1 属于显式 old class IDs 的像素成为候选，`255` 永不参与。
- `fixed / batch_global / batch_class / artifact_class` 四类阈值策略。
- `margin_min`、`min_conf/max_conf`、`min_pixels`、`shrinkage`。
- per-class fallback 记录，区分 `none/global/fixed/artifact/fixed_empty`。
- artifact schema 校验：dataset/task/step/setting、teacher sha256、old class IDs、阈值超参、loss type、random seed、可选 max_batches。

### 2.2 Trainer 接入

修改：

```text
trainer/trainer.py
```

实现内容：

- 新增协议 guard：`setting == sequential` 且伪标签启用时直接报错。
- step1 teacher 明确使用 step0 DeepLab checkpoint；step2+ teacher 使用上一步 AIR checkpoint。
- 在每个 incremental `AIR.fit(X, y)` 前应用伪标签，不改 Dataset，不影响 eval/test。
- 训练后输出：

```text
checkpoints/<subpath>/voc/<task>/<setting>/step<k>/pseudo_label_stats.json
```

统计字段包括 candidate/accepted 总数、接受比例、per-class candidates/accepted、mean thresholds、fallback counts、teacher checkpoint/hash。

注意：`teacher_class_count_for_step()` 当前使用 `sum(opts.num_classes[:curr_step + 1])` 是正确的，因为本项目 `num_classes` 在 `train.py` 中编码为 `[1, step0_fg, step1_fg, ...]`。例如 VOC 15-5 step1 的 teacher 是 step0 DeepLab，输出类数为 `1 + 15 = 16`。

### 2.3 参数与 manifest

修改：

```text
utils/parser.py
utils/run_manifest.py
run.sh
```

新增参数：

```text
--pseudo_label_strategy off|fixed|batch_global|batch_class|artifact_class
--pseudo_label_quantile
--pseudo_label_min_conf
--pseudo_label_max_conf
--pseudo_label_min_pixels
--pseudo_label_shrinkage
--pseudo_label_margin_min
--pseudo_label_threshold_artifact
--pseudo_label_threshold_max_batches
--pseudo_label_stats
```

兼容规则：

- 未启用 `--use_pseudo_label` 时，真实策略解析为 `off`。
- 旧命令 `--use_pseudo_label --pseudo_label_confidence 0.7` 映射为 `fixed`。
- 新字段已写入 `run_manifest.json` 的 `args` 和 flat keys。

### 2.4 工具脚本

新增：

```text
tools/run_adaptive_pseudo_label.sh
tools/calibrate_pseudo_label_thresholds.py
tools/summarize_adaptive_pseudo_label.py
```

作用：

- `run_adaptive_pseudo_label.sh`：专用 runner。step0 强制不启用伪标签，step1 才启用策略；`SKIP_STEP0=1` 时必须显式提供 `BASE_SUBPATH`。
- `calibrate_pseudo_label_thresholds.py`：生成 `artifact_class` 所需 per-class threshold artifact。
- `summarize_adaptive_pseudo_label.py`：读取最新 `run_manifest*.json`、`pseudo_label_stats.json` 和最新 `test_results_*.json`，生成 markdown 摘要。

## 4. Code Review

CodeRabbit CLI：

```text
coderabbit 0.6.3
```

执行：

```bash
coderabbit review --agent -t uncommitted
```

处理过的有效问题：

- `--bins <= 0` 缺少校验。
- calibration device 没有处理 `--gpu_id` / `CUDA_VISIBLE_DEVICES`。
- 伪标签路径 dtype 转换不够集中。
- `artifact_class` 缺 class threshold 时原本可能变成裸 `KeyError`。
- `SKIP_STEP0=1` 没有强制 `BASE_SUBPATH`。
- summary 选择最新 result/manifest 的逻辑不够稳。
- batch quantile 直接在 CUDA 大 tensor 上计算，已改为 CPU quantile，降低 CUDA 内存风险。
- artifact 元数据缺少 `loss_type/random_seed/max_batches`。
- summary 对不存在的输出目录应早期失败。

最终复核：

```text
CodeRabbit review_completed, findings: 0
```

人工判定的误报：

- “teacher class count 应排除 curr_step”这条不适用于本项目。由于 `num_classes=[1, step0_fg, step1_fg, ...]`，当前 step1 teacher 类数应取 `[:curr_step + 1]`，即 `[1, 15] -> 16`。
