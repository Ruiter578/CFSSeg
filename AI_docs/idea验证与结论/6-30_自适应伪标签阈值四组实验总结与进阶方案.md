# 自适应伪标签阈值四组实验总结与进阶方案

生成时间：2026-06-30
代码分支：`feature/adaptive-pseudo-label`
实验协议：VOC `15-5 overlap step1`，`deeplabv3_resnet101`，batch size 32，seed 1，`buffer=8196`，`gamma=1`
共同 step0：`20260627_pseudo_15-5_overlap_batchclass_q0p7_seed1_bs32`

## 1. 最终结论

第一轮四组实验已经跑完，结论比前一轮更明确：

1. **伪标签本身有效，但收益不大**：`fixed0.7` 相比 `off` 提升 `+0.4292` all mIoU，说明在 `15-5 overlap` 中恢复旧类背景像素确实有正向作用。
2. **当前自适应 q0.7 策略失败**：`batch_global q0.7` 几乎等于 `off`，`batch_class q0.7` 只小幅好于 `off`，两者都明显低于 `fixed0.7`。
3. **失败主因不是代码没跑通，而是策略定义过保守**：`q=0.7` 的 quantile 策略等价于只保留 top 30% 候选。当前固定阈值 `0.7` 接受约 90.35% 候选，反而效果最好。
4. **“自适应”的定义需要重做**：不能再把“每 batch 取 70% 分位点”作为默认自适应方案。后续应转向“接受比例校准、paper-equivalent 阈值、离线校准 artifact、伪标签质量 audit、置信度/边际联合门控”等更完整路线。
5. **不能把 `15-5 overlap` 结果直接和论文 `15-1 overlapped 69.36` 对比**：要和 69.36 对比，必须完整跑 VOC `15-1 overlapped` 到 step5，并使用明确的 paper-equivalent 配置。

## 3. 第一性原理解释

### 3.1 当前协议到底在测什么

`15-5 overlap step1` 的训练标签只保留当前新类 `16-20`，旧类 `1-15` 会被映射为 background。伪标签的作用不是改模型结构，而是在 AIR fit 之前把部分“当前标签为 background、teacher 高置信预测为旧类”的像素改回旧类标签。

因此它真正改变的是：

```text
AIR 闭式 fit 的 label target matrix
```

而不是：

```text
backbone / DeepLab teacher / test label / metric 计算
```

### 3.2 为什么 fixed0.7 最好

当前 teacher 的候选旧类像素置信度整体很高：

```text
fixed0.7 接受 45,141,859 / 49,963,137 = 90.3503%
```

这说明大量候选都超过 0.7。对于 `15-5 step1` 这个最后一步，背景里的主要矛盾是旧类被隐藏成背景，而不是未来类污染。此时高召回地恢复旧类像素更重要，固定阈值 0.7 的“多接受”反而成为优势。

### 3.3 为什么 batch_global q0.7 几乎无效

`batch_global q0.7` 每个 batch 取候选置信度 70% 分位点，保留 top 30%。实际统计：

```text
accepted_ratio = 30.0003%
mean global threshold ≈ 0.99666
```

这相当于只相信极端高置信的旧类预测。它几乎不改变 AIR 的训练 target，所以 all mIoU 只比 `off` 高 `+0.0166`。

对抗性解释：如果 batch_global 这么保守仍然没明显坏掉，说明它引入的噪声少；但它没有足够召回，修复 semantic drift 的力度不够。

### 3.4 为什么 batch_class q0.7 比 batch_global 稍好

`batch_class q0.7` 也是总接受约 30%，但按类别分配接受名额。它避免了全局阈值被 class 15 等高置信大类支配，使 class 2、9、11 等类获得更多伪标签，因此比 batch_global 多 `+0.128` all mIoU。

但是它仍低于 fixed0.7，因为它在每个类上都强行只保留约 top 30%。这不是“类别自适应做错了”，而是“当前自适应定义把召回压得太低”。

## 4. 关键 per-class 证据

## 5. 对抗性审查

### 5.1 当前能确定的事实

已由 manifest / stats / result 文件确认：

- 四组实验使用同一个 step0 checkpoint hash：

```text
6505c26b567cefbb9eb099af174961cbd4596d139b30a7f92f952ad494a9a913
```

- 三组启用伪标签的 candidate count 完全一致：

```text
49,963,137
```

- `batch_global q0.7` 与 `batch_class q0.7` 的 accepted ratio 都约 30%。
- `fixed0.7` 的 accepted ratio 约 90.35%。
- 当前结果不是由 batch size、checkpoint、setting、teacher 或 eval script 差异造成的。

### 5.2 如果当前结论是错的，最可能错在哪里

| 可能错误 | 风险 | 反证方式 |
| --- | --- | --- |
| 单 seed 偶然性 | 中 | 在候选 top2 策略上补 seed2/3，不全矩阵重复 |
| `15-5 overlap` 太特殊 | 高 | 跑 `15-1/10-1 overlapped` full protocol |
| fixed0.7 不是 paper-equivalent | 高 | 补 fixed0.6，因为论文 `tau=0.4` 约等于 confidence `0.6` |
| 没有伪标签质量证据 | 高 | 写 raw-mask audit，计算 pseudo precision/recall/confusion |
| 当前“自适应”定义太窄 | 高 | 扩展为低 q、min/max clip、offline artifact、margin、weighted pseudo label |

## 6. 原论文结果与比较口径

论文 2D VOC 结果中，常被引用的 overlap 结果是：

| protocol | paper Ours old | paper Ours new | paper Ours all |
| --- | ---: | ---: | ---: |
| `15-1 overlapped` | 79.16 | 38.00 | **69.36** |
| `10-1 overlapped` | 75.02 | 41.20 | **58.91** |

同时，本项目文档记录论文 2D 实现配置为：

```text
DeepLabV3 + ResNet101
step0 train epoch = 50
batch size = 32
BCE loss
d_E = 8192
gamma = 1
tau = 0.4
```

必须注意：论文 `tau=0.4` 是不确定性阈值。论文定义近似为：

```text
U = 1 - confidence
U <= tau  <=>  confidence >= 1 - tau
```

因此 `tau=0.4` 对当前代码的置信度阈值约等于：

```text
pseudo_label_confidence = 0.6
```

这意味着：

1. 当前 `fixed0.7` 是代码默认固定阈值强 baseline。
2. 若要对齐论文，应补 `fixed0.6`。
3. 只有完整跑 `15-1 overlapped` 到 step5，才能和 69.36 对比。
4. `15-5 overlap` 的 70.7383 不能写成超过论文 69.36，因为协议不同。

参考：

- CFSSeg arXiv 页面：<https://arxiv.org/html/2412.10834v2>
- `AI_docs/论文精读/CFSSeg_精读笔记.md`
- `AI_docs/代码理解与学习/CFSSeg与SegACIL实验配置对照报告.md`

## 7. 重新定义“自适应”

当前 `batch_global/batch_class q0.7` 只实现了“按分位数控制接受比例”。这是一种自适应，但不是最适合当前任务的自适应。

后续应至少区分四类自适应：

### 7.1 接受比例自适应

通过 quantile 控制每 batch 或每类接受比例。

候选：

```text
q = 0.1 / 0.3 / 0.5 / 0.7
```

含义：

```text
q=0.1 -> 接受 top 90%
q=0.3 -> 接受 top 70%
q=0.5 -> 接受 top 50%
q=0.7 -> 接受 top 30%
```

当前结果已经证明 `q=0.7` 太保守，但没有证明 quantile 自适应整体无效。

### 7.2 paper-equivalent 阈值校准

补 `fixed0.6`，对应论文 2D `tau=0.4` 的置信度阈值近似。它不一定是新方法，但必须作为严肃 baseline。

### 7.3 离线 artifact 自适应

先在 step1 训练集上用 teacher 扫一遍候选分布，生成固定 artifact，再训练时读取该 artifact。优势：

- 阈值不随 batch 抖动；
- 可记录 teacher hash；
- 可做 per-class threshold；
- 更适合论文复现与跨服务器同步。

### 7.4 质量感知自适应

不只看 top1 confidence，还引入：

- top1-top2 margin；
- 类别 candidate cap；
- per-class quota；
- pseudo label sample weight；
- soft label 或 confidence-weighted label matrix；
- raw-mask audit 后的 class-specific correction。

这类方案改动更大，但更符合“自适应”的论文价值。

## 8. 不保守的后续实验路线

后续不应只启动一个最小 next run。建议采用“三线并行、分阶段筛选”：

```text
保守验证线：确认 paper-equivalent 与 fixed baseline
进取探索线：系统搜索 adaptive 策略和参数
论文证据线：对齐 15-1 / 10-1 overlapped，与 69.36 / 58.91 对比
```

### Phase A：15-5 overlap 搜索矩阵

复用当前 step0，补齐参数搜索。优先跑下面 10 组：

| 组别 | strategy | 参数 | 目的 |
| --- | --- | --- | --- |
| A1 | `fixed` | confidence `0.6` | paper-equivalent tau=0.4 |
| A2 | `fixed` | confidence `0.8` | 判断 fixed0.7 是否最优 |
| A3 | `fixed` | confidence `0.9` | 高精度低召回边界 |
| A4 | `batch_global` | q `0.1` | top 90%，接近 fixed0.7 接受量 |
| A5 | `batch_global` | q `0.3` | top 70% |
| A6 | `batch_global` | q `0.5` | top 50% |
| A7 | `batch_class` | q `0.1` | 类别均衡 + 高召回 |
| A8 | `batch_class` | q `0.3` | 类别均衡 + 中高召回 |
| A9 | `batch_class` | q `0.5` | 类别均衡 + 中等召回 |
| A10 | `batch_class` | q `0.3`, `min_conf=0.6` | 防止低质候选被 quota 强行接受 |

筛选标准：

```text
必须超过 fixed0.7 的 70.7383，才算 adaptive 在 15-5 overlap seed1 上成立。
如果只超过 off 但不超过 fixed0.7，只能说明伪标签有效，不能说明 adaptive 有效。
```

### Phase B：raw-mask pseudo audit

这不是训练实验，而是诊断工具。需要新增工具：

```text
tools/audit_pseudo_label_quality.py
```

功能：

1. 对 overlap step1 train loader 同时读取隐藏后的训练 label 和原始完整 GT。
2. 用 step0 teacher 生成候选 pseudo labels。
3. 对 fixed0.6/fixed0.7/fixed0.8/batch_global/batch_class 等策略计算：
   - pseudo precision；
   - pseudo recall；
   - per-class confusion；
   - accepted ratio；
   - old-class recovery；
   - false old on true background；
   - false old on future/current class。

为什么这一步必要：当前 fixed0.7 最好，但我们不知道它是因为 pseudo precision 足够高，还是因为高 recall 抵消了噪声。没有 audit，就无法设计更强 adaptive。

### Phase C：15-5 disjoint 复验

选择 Phase A 的 top2：

```text
fixed0.7 或 fixed0.6
best adaptive
off
```

跑 `15-5 disjoint`。目的不是论文主表对比，而是验证 non-sequential 另一种背景污染协议下是否稳定。

### Phase D：15-1 overlapped 论文对齐

这是回答 69.36 的关键。

必须完整跑：

```text
TASK=15-1
SETTING=overlap
START_STEP=0
END_STEP=5
```

建议矩阵：

| 组别 | 策略 | 目的 |
| --- | --- | --- |
| D0 | paper reproduction / fixed0.6 | 对齐论文 tau=0.4，目标接近 69.36 |
| D1 | fixed0.7 | 当前代码强 baseline |
| D2 | best adaptive from Phase A/B | 新方法候选 |
| D3 | off | 伪标签消融，证明 semantic drift 修复必要性 |

判定：

```text
D2 > D1 且 D2 > 69.36：可以主张 adaptive pseudo-label 在论文 overlap 协议上超过原 CFSSeg。
D2 > D1 但 D2 <= 69.36：本地实现强于本地 baseline，但未超过论文报告。
D1 或 D0 已显著低于 69.36：先解决复现 gap，再讨论新方法。
```

### Phase E：10-1 overlapped 压力测试

`10-1` 有 10 个增量 step，semantic drift 更长、更难。论文 Ours all mIoU 是 `58.91`。

建议只跑 Phase D 中最有希望的两组：

```text
fixed0.6 或 fixed0.7
best adaptive
```

必要时再补 `off`，用于证明伪标签消融。

## 9. 代码实现建议

当前核心训练代码已经能跑 fixed/batch_global/batch_class，但要支撑不保守的实验矩阵，建议新增三个工具，而不是手写几十条 tmux 命令：

### 9.1 grid runner

新增：

```text
tools/run_pseudo_label_grid.sh
```

功能：

- 接收一个 CSV/TSV 或内置数组；
- 每行定义 `SUBPATH / TASK / SETTING / START_STEP / END_STEP / STRATEGY / CONFIDENCE / QUANTILE / MIN_CONF / BASE_SUBPATH`；
- 自动检查输出目录是否已存在；
- 每组实验写独立 log；
- 支持只打印命令 dry-run；
- 支持 A100/TRS 分队执行。

### 9.2 grid summarizer

新增：

```text
tools/summarize_pseudo_label_grid.py
```

功能：

- 扫描一批 subpath；
- 汇总 final step 的 all/old/new mIoU；
- 汇总 accepted ratio、candidate count、teacher hash；
- 输出 markdown + csv；
- 标记是否超过 fixed0.7、fixed0.6、paper baseline。

### 9.3 raw-mask audit

新增：

```text
tools/audit_pseudo_label_quality.py
```

这是后续方法设计的关键，不只是辅助脚本。它决定 adaptive 应该追求：

- 更高 recall；
- 更高 precision；
- 类别均衡；
- 边界保守；
- pseudo sample weighting。

## 10. 初步论文叙事判断

当前结果不能支持：

```text
“batch-class adaptive threshold improves CFSSeg.”
```

当前结果可以支持：

```text
“Pseudo-labeling is still useful in VOC 15-5 overlap, but naive batch quantile adaptation is too conservative. A stronger adaptive method should calibrate acceptance ratio and per-class quality rather than blindly taking top 30% candidates.”
```

如果后续 Phase A/B 找到超过 fixed0.7 的 adaptive，并且 Phase D 在 `15-1 overlapped` 超过 69.36，则可升级为论文贡献：

```text
Quality-Calibrated Pseudo-Labeling for Analytic Class-Incremental Segmentation
```

如果最终没有超过 fixed baseline，则伪标签线应降级为辅助消融，把主线资源转回 RHL / 集成学习。

## 11. 下一步建议

不要立即跑 `15-1` 大实验。先补 Phase A 的搜索矩阵和 Phase B 的 audit 工具。理由：

1. 现在还不知道 `fixed0.7` 是否就是最优 fixed baseline。
2. 现在也不知道低 q adaptive 是否能恢复高召回并超过 fixed。
3. 如果直接跑 `15-1/10-1`，成本高且失败后不可解释。

推荐下一轮执行顺序：

1. 实现 `tools/run_pseudo_label_grid.sh` 和 `tools/summarize_pseudo_label_grid.py`。
2. 跑 Phase A 的 10 组 `15-5 overlap` 搜索。
3. 实现并运行 raw-mask audit，解释 fixed/best adaptive 的伪标签质量。
4. 选出 top2，跑 `15-5 disjoint`。
5. 再跑 `15-1 overlapped` full protocol，对比 69.36。
6. 最后视结果决定是否跑 `10-1 overlapped`。

这条路线比“再随便跑一个 q 值”更进取，但仍然可控：先用便宜的 `15-5 step1` 找机制，再把最有希望的方案带到论文协议。
