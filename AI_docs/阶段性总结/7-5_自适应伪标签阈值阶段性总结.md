当前最强单点是：

```text

all mIoU = 0.708081

```

它高于当前同协议最强 fixed baseline `fixed0.6 = 0.707731`，但绝对提升只有约 `+0.000350` all mIoU。结论应写成“低分位 offline artifact 有稳定弱正向信号”，不能写成“已经形成论文级显著涨点”。

## 4. 机制解释

从第一性原理看，当前 `15-5 overlap step1` 的主要问题不是“旧类伪标签太脏”，而是“旧类像素被隐藏为 background 后监督不足”。因此：

1. 接收率太低会损失旧类监督，q0.7 的 top 30% 策略明显过保守。
2. `fixed0.6` 比 `fixed0.7/0.8/0.9` 强，说明更高召回更重要。
3. `artifact q0.005/q0.01` 比 `fixed0.6` 略强，说明按旧类分布自适应地放宽低置信类别有一点价值。
4. `min_conf=0.6` 会把这种低阈值优势抹掉，结果退回 fixed0.6 附近。

当前涨点主要发生在 old 0-15，new 16-20 只有小幅跟随提升。这符合伪标签保护旧类的机制预期。

## 6. 尚未完成的部分

### 6.1 raw-mask quality audit

仍未实现。它用于回答：

- artifact q0.005/q0.01 的伪标签 precision/recall 到底是多少；
- 哪些旧类依赖低于 0.6 的阈值；
- 低阈值是否引入了明显错误，还是主要补回被隐藏的旧类像素。

这不是训练必需项，但对论文解释和后续方法设计很关键。

### 6.2 15-5 disjoint 复验

建议最小矩阵：

| 组别 | 目的 |
|---|---|
| off | disjoint baseline |
| fixed0.6 | 最强固定阈值 baseline |
| artifact q0.005 或 q0.01 | 当前 best adaptive |

预计 3 个 step1 实验。如果没有可复用 disjoint step0，还需要先跑一个 disjoint step0。

### 6.3 15-1 / 10-1 overlap 论文协议

这是论文证据线。建议先做 `15-1 overlap`：

| 组别 | 目的 |
|---|---|
| off | 本地无伪标签 baseline |
| fixed0.6 | 论文 tau=0.4 的近似等价固定阈值 |
| fixed0.7 | 当前代码历史固定阈值 |
| artifact q0.005 或 q0.01 | 当前 best adaptive |

`15-1` 有 step1-step5，工作量明显大于 `15-5`。如果每组都从 step0 到 step5 跑完整链路，最少约 4 组完整多步实验。若先做筛选，可只跑 step1/step2 早停判断，再决定是否跑全。

### 6.4 多 seed / 显著性

当前所有核心结果主要是 seed1，同协议内变量控制干净，但涨点很小。若要写成论文贡献，至少需要：

- 在 `15-5 overlap` 对 `fixed0.6` 与 `artifact q0.005/q0.01` 做 seed1/2/3 paired 复验；
- 或者在 `15-1 overlap` 主协议上证明多步累计收益明显大于当前 0.035 个百分点级别。

## 8. 当前建议

短期建议不要继续在 `15-5 overlap` 上无限细化 q 值。q0.005/q0.01/q0.015/q0.00 已经把边界摸清：

```text
最佳区间：q0.005 - q0.01
机制关键：极高接收率、允许部分类阈值低于 0.6
收益强度：稳定但很小
```

下一步应转向：

1. raw-mask audit，解释弱正向信号；
2. `15-5 disjoint` 复验，确认适用边界；
3. `15-1 overlap` 论文协议，判断是否值得继续作为论文方法主线。

如果这三步没有放大收益，自适应伪标签阈值应降级为辅助消融，主线资源转回 RHL 和轻量解析集成。
