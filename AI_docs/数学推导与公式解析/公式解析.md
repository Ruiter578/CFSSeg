先给结论：你卡住的其实是两个层面。第一个是公式里的“矩阵索引”没有吃透。第二个是 CSS（Class-Incremental Semantic Segmentation，类别增量语义分割）里“当前步的数据里到底有哪些类出现、哪些类被标注、哪些类被压成背景”这三件事混在一起了。把这两层拆开，你就会一下清楚很多。CFSSeg 里对分割目标和三种设置的原始定义就在背景部分。

先讲最后一个公式。论文先定义  
(q_\theta: X \to \mathbb{R}^{N \times |C|})，意思是：模型对一个输入 (x) 的输出，不是一个单独类别，而是一个二维表。这个表有 (N) 行、(|C|) 列。(N) 是元素个数，也就是图像里的像素数，或者点云里的点数；(|C|) 是类别总数，包括背景类。论文明确说 (q_\theta(x)[i,c]) 表示“第 (i) 个元素属于第 (c) 类的概率”。也就是说，([i,c]) 就是一个标准的矩阵索引：第 (i) 行、第 (c) 列。

所以你可以把它想成下面这张概率表：

[  
q_\theta(x)=  
\begin{bmatrix}  
p(\text{pixel }1 \to \text{bg}) & p(\text{pixel }1 \to \text{cat}) & p(\text{pixel }1 \to \text{dog}) & \cdots \  
p(\text{pixel }2 \to \text{bg}) & p(\text{pixel }2 \to \text{cat}) & p(\text{pixel }2 \to \text{dog}) & \cdots \  
\vdots & \vdots & \vdots & \ddots  
\end{bmatrix}  
]

这里第 (i) 行对应第 (i) 个像素或点，第 (c) 列对应某个类别。于是  
[  
\hat y_i=\arg\max_{c\in C} q_\theta(x)[i,c]  
]  
表示：对第 (i) 个像素，选概率最大的那个类别作为预测标签。

严格地写，应该是“对每个 (i)，都有一个 (\hat y_i)”。论文里写成  
[  
\hat y=\arg\max_{c\in C} q_\theta(x)[i,c]  
]  
本质上是省略写法，默认你知道这是“逐像素逐点进行 argmax”。更严谨的写法是：  
[  
\hat y = (\hat y_1,\hat y_2,\ldots,\hat y_N),\quad  
\hat y_i=\arg\max_{c\in C} q_\theta(x)[i,c].  
]  
这一点很重要，因为分割不是给整张图一个类，而是给每个位置一个类。

下面讲 CSS 的三种设置。你现在最容易混淆的是“数据中出现了什么”和“标注里写了什么”是两回事。论文里对三种设置的定义是：Sequential 里旧类和新类都被标注；Disjoint 里旧类在当前任务中被统一标成背景；Overlapped 更难，因为当前只标新类和背景，但背景里可能混有真实背景、旧类，甚至未来类。

我用一个非常具体的例子讲。假设：

第 1 步学过旧类：猫、狗。  
第 2 步要学新类：自行车。  
当前第 2 步的一张图里其实同时出现了：狗、自行车、天空、汽车。  
其中汽车是未来第 3 步才会学的类。

第一种，Sequential Setting。这里最理想。当前这张图里，狗会标成“狗”，自行车标成“自行车”，天空标成“背景”或其真实类别，总之旧类和新类都能拿到正确标签。也就是说，监督信号没有故意“误伤”旧类。论文就是这么定义的：每一步都同时有旧类别和新类别的标签。

这意味着什么？意味着模型在学新类时，不会被迫把旧类当背景，因此几乎没有“标签层面”的冲突。它更像一种较温和的增量设置。

第二种，Disjoint Setting。这里开始难了。当前步虽然数据里仍然可能出现旧类和新类，但标注时只认真标新类；凡是以前学过的旧类，在当前任务里统统压成背景。CFSSeg 就是这样定义的：之前学过的类别在当前任务会被标成背景，这会导致 semantic drift。

还是上面的例子。第 2 步这张图里有“狗”和“自行车”。在 disjoint 标注下：  
狗像素不再写“狗”，而是写“背景”；  
自行车像素写“自行车”。

这时，如果模型其实把狗预测成“狗”，从人类角度看它没错，但从当前步标签角度看，它反而“错了”，因为标签说那是背景。于是梯度会惩罚“狗”这个旧类响应，把它往背景里推。

这就是 semantic drift，语义漂移。它不是说特征凭空漂了，而是说：由于监督标签把旧类错误地并入背景，模型被训练得逐渐把“旧类语义”往“背景语义”方向移动。CFSSeg 还明确说，在 disjoint 和 overlapped 设置里，semantic drift 会出现，表现为先前学过的类别在新数据中塌缩为背景标签。

你可以把它直觉地理解成一句话：模型不是忘了旧类，而是训练信号在逼它把旧类当背景。

第三种，Overlapped Setting。这个比 disjoint 更毒。论文定义是：当前只标新类和背景，但这个“背景”并不纯，里面可能同时混有真实背景、旧类、未来尚未引入的类。

还用刚才例子。当前图里有狗、自行车、天空、汽车。overlapped 标注可能是：

自行车 → 标成“自行车”  
天空 → 背景  
狗 → 背景  
汽车 → 背景

注意这里背景已经是个“大杂烩”了：

真正背景：天空、路面之类  
旧类：狗  
未来类：汽车

也就是说，background 这个标签不再对应一个相对稳定的语义簇，而是一个混合垃圾桶。这个时候模型面对“背景”监督时，根本不知道你是想让它学会“非目标区域”，还是想让它压制旧类，还是想暂时忽略未来类。监督信号会非常脏。

这也是为什么 overlapped 比 disjoint 更难。disjoint 里至少你主要是在“旧类 vs 新类”之间冲突；overlapped 里则是“真实背景 vs 旧类 vs 未来类 vs 新类”全部搅在一起。后来的工作也会把 overlapped 解释成更真实更困难的场景：数据里不但有旧类和当前新类，还可能有未来类，而标签仍然只给当前新类和背景。

你可以把三种设置压缩成一句最核心的话：

Sequential：出现什么，就尽量如实标什么。  
Disjoint：旧类出现了，但故意标成背景。  
Overlapped：旧类和未来类都可能出现，而且都一起被塞进背景。

所以从难度上通常是：  
Sequential < Disjoint < Overlapped。这个排序的根本原因不是模型结构，而是监督噪声越来越大，尤其背景标签越来越“不干净”。CFSSeg 也正是因此专门在 disjoint 和 overlapped 里引入了基于不确定性的 pseudo-labeling（伪标签）去缓解 semantic drift。

再进一步，把“semantic drift”讲得更机制一点。

为什么“旧类被标背景”会伤害旧类？

因为分割训练通常是逐像素交叉熵。对一个本来属于旧类“狗”的像素，如果当前标签写成背景，那么损失函数会推动：

背景 logit 变大  
狗 logit 变小

这种更新不是偶然噪声，而是系统性的、反复发生的。所以随着训练步数增加，模型越来越倾向于把旧类区域解释成背景。于是你在测试旧类时会发现：不是模型完全没有特征了，而是分类头或决策边界把旧类吞进背景了。这就是“漂移”的本质。

如果再叠加 overlapped setting，未来类也被塞进背景，那模型会学到一个极其宽泛的背景原型：凡是现在没标成新类的很多东西，都先归背景。这样后面未来类真正登场时，它们的表征往往已经被预先压扁到背景区域了，学习会更难。这也是很多 CISS 工作反复处理 background shift、background entanglement、semantic drift 的原因。

最后我帮你把这段内容整理成一个最适合记忆的版本：

你看到的最后一个公式里，(q_\theta(x)[i,c]) 就是“第 (i) 个像素或点属于第 (c) 类的概率”，([i,c]) 是矩阵索引，不是什么新的算子。分割预测就是对每个 (i) 单独做一次 argmax。

CSS 三种设置的本质区别在于：当前步的数据中虽然都可能出现旧类、新类，甚至未来类，但标签怎么给不一样。Sequential 中旧类和新类都正确标注；Disjoint 中旧类被压成背景，于是产生 semantic drift；Overlapped 中背景不仅包含真实背景，还混入旧类和未来类，使背景标签变成高度混杂的监督源，因此更难。

如果你愿意，我下一条可以继续给你画一个“15-1 增量分割”的具体标注示意图，用一个 4×4 小 mask 把 sequential、disjoint、overlapped 三种标注方式逐格画出来，你会彻底不混。



可以把它一句话理解成：

CFSSeg 里的伪标签（Pseudo-label，伪造标签/预测标签）不是“给新类补标签”，而是**在增量步骤里，把当前标注成背景的一部分像素或点，重新改写成“上一阶段模型认为的旧类标签”**，从而避免旧类被背景吞掉，也就是缓解 semantic drift（语义漂移）。

先说它“是什么”。

论文在方法总览里写得很直接：在第 (t) 步，先用第 (t-1) 步的模型生成 pseudo labels（伪标签），再把这些伪标签和当前数据的 ground truth（真实标注）合成 mixed labels（混合标签），然后再用这些 mixed labels 去更新当前步的分类头。也就是说，CFSSeg 的训练标签不是只靠当前数据集给的原始标注，而是“原始标注 + 上一阶段模型提供的补充标签”共同组成。

这背后的动机非常明确。论文指出，在 disjoint setting（分离设置）和 overlapped setting（重叠设置）中，旧类在当前任务里会被标成背景，这会造成 semantic drift，也就是模型逐渐把旧类往背景类上压塌。CFSSeg 专门引入了一个基于 uncertainty（不确定性）的 pseudo-labeling strategy（伪标签策略）来缓解这个问题。

你可以先用一个极简例子抓住直觉。

假设第 1 步学过“狗”，第 2 步学“自行车”。第 2 步的一张图里同时有狗和自行车，但在 disjoint 或 overlapped 设置下，标注里往往只有“自行车”和“背景”，狗会被压成背景。此时如果你直接拿这个标签训练，那么狗区域会收到“你应该是背景”的监督，模型就会慢慢忘掉“狗”。CFSSeg 的伪标签做的事是：让旧模型先看这张图，如果它对某个背景像素很有把握地认为“这是狗”，那就别再把它当背景训练了，而是把这个背景像素改写成“狗”。这样，旧类不会被错误监督持续压向背景。

所以伪标签的核心功能不是“增强数据”，而是**纠正当前步标签中的系统性错误背景**。

下面讲 2D 图像里它到底怎么做。

论文先定义了一个 uncertainty（不确定性）：  
[  
U_i = 1 - \sigma\left(\max_c q_{\theta_{t-1}}(i,c)\right)  
]  
这里 (q_{\theta_{t-1}}(i,c)) 是上一阶段模型对第 (i) 个像素属于类别 (c) 的 logit（未归一化输出），(\max_c) 表示取所有类别里最大的那个分数，(\sigma) 是 Sigmoid（S 形激活函数）。然后 (U_i) 越小，表示上一阶段模型越自信；(U_i) 越大，表示越不确定。

这点你可以直觉化地看：  
如果旧模型对某像素的最大 logit 很大，说明它很确信“这像素就是某个类”，那 Sigmoid 后接近 1，于是  
[  
U_i \approx 0  
]  
也就是“不确定性低，可信”。

接着论文给出 2D 的伪标签规则：  
如果当前像素的真实标签本来就是新类 (S_t) 里的某一类，那就保留当前真实标签；  
如果当前像素被标成背景 (c_b)，并且旧模型对它不够自信，也就是 (U_i > \tau)，那就继续保留背景；  
如果当前像素被标成背景 (c_b)，但旧模型对它很自信，也就是 (U_i \le \tau)，那就把它改成上一阶段模型的预测标签 (y_i^{t-1})。

把它翻成更易懂的话就是：

“新类标签绝对信 ground truth；背景标签不一定真信，要先问旧模型一句：你觉得这真是背景吗？如果旧模型很有把握说不是背景，而是某个旧类，那我就把它改成那个旧类。”

这一步非常关键，因为在 disjoint 和 overlapped 里，**背景标签本来就不干净**。伪标签等于对背景做一次“旧类回收”。

再讲它“起什么作用”。

第一，最直接的作用是缓解 semantic drift（语义漂移）。论文明确说，这个方法用于减轻 disjoint 和 overlapped 下的 semantic drift。

第二，它能让旧类保留下来，不被背景吞掉。消融实验也验证了这一点：在 Pascal VOC2012 的 overlapped 10-1 设置下，去掉伪标签之后，旧类 mIoU（mean Intersection over Union，平均交并比）从 75.02 降到 71.83，新类从 41.20 降到 36.19，总体从 58.91 降到 54.86。论文据此认为 pseudo-labeling 在防止 semantic drift 上起了关键作用。

这组结果很重要，因为它说明伪标签不只是“保老类”，它连新类也有帮助。为什么？因为如果背景里混着一堆旧类却被错误当成背景，新的分类边界也会学歪。把旧类从背景里分离出来后，背景分布更干净，新类学习也会更稳定。

第三，它和 CFSSeg 的 closed-form solution（闭式解）是配套的。CFSSeg 不靠多轮梯度下降更新，而是冻结编码器，只更新解析形式的分类头。此时训练信号本身就更关键。如果标签里大量旧类被错标成背景，闭式解一样会忠实地把这种错误监督写进参数里。所以伪标签在这里相当于**先修标签，再做闭式更新**。这也是为什么图 2 里伪标签发生在分类头更新之前。

下面我再把“为什么要加 uncertainty（不确定性）阈值”讲透一点。

你可能会想，既然旧模型能预测旧类，那所有背景都直接改成旧模型预测不就好了？不行。因为旧模型也会犯错。如果无脑相信旧模型，就会把真实背景错改成旧类，或者把未来类错改成旧类，这会引入新的噪声。于是论文用了一个阈值 (\tau) 去筛选：只有当旧模型足够自信时，才接管背景像素。否则宁可保持背景不动。

所以这其实是一个“保守纠错”机制：  
只修那些“高置信度疑似旧类”的背景；  
对不确定区域不乱修。

这个设计很合理，因为 overlapped setting 最大的问题就是背景是混合桶，里头既有真实背景，也有旧类，还有未来类。伪标签如果不做置信度控制，很容易把这个桶搅得更乱。

再说 3D 点云（Point Cloud，点云）里的伪标签，它比 2D 多了一步空间邻域利用。

论文在 3D 里没有直接沿用 2D 的简单置信度，而是结合 KNN（K-Nearest Neighbors，K 近邻）和 MC-dropout（Monte Carlo Dropout，蒙特卡罗随机失活）做 uncertainty estimation（不确定性估计），并用 BALD（Bayesian Active Learning by Disagreement，基于分歧的贝叶斯主动学习）作为不确定性准则。

3D 的规则大意是：

如果当前点是真实新类，保留 ground truth；  
如果当前点被标成背景，但旧模型以前把它 confidently（高置信度地）预测成某个非背景旧类，那就直接用旧标签；  
如果旧模型对这个点本身不够确定，或者旧模型也给它背景，那就去看它的近邻里有没有高置信度、非背景的邻居标签，可以借邻居标签补；  
如果还是找不到可靠邻居，就保留背景。

为什么 3D 要多这一步邻域传播？因为点云是离散采样、局部稀疏、噪声更强，单点预测抖动更大。引入空间邻域，相当于利用“附近点大概率属于相似结构”的几何先验，让伪标签更稳。

现在我把整件事压成一个你最该记住的版本：

CFSSeg 的伪标签，本质上是“**用旧模型纠正当前任务里被错误标成背景的旧类区域**”。它只在 disjoint 和 overlapped 这类背景不干净的设置里特别重要。具体做法是：先用旧模型给当前样本出预测，再根据不确定性决定是否信任这个预测；如果某个当前背景像素或点被旧模型高置信度地判成旧类，就把它从背景改回旧类，和原始标注一起构成 mixed labels，用来更新当前步模型。这样做的主要功能是抑制 semantic drift，减少旧类塌缩到背景，同时让背景监督更干净，从而也帮助新类学习。

最后给你两个你读论文时很值得注意的小点。

第一，论文公式里的 (U_i) 虽然叫 uncertainty，但它实际上是由“最大 logit 经过 Sigmoid 再取 (1-)”得到的，所以本质上接近“逆置信度”。也就是数值越小越可信，越大越不可信。这个方向别看反了。

第二，伪标签不是万能的。它强依赖旧模型质量。如果旧模型本来就没学好旧类，或者对 domain shift（领域偏移）很敏感，那伪标签也会带偏训练。所以它的收益通常在“旧模型还能较好识别旧类，但当前标签把旧类错压成背景”的场景最明显。这一点虽然论文没有单独展开写限制，但从机制上是很自然的推论。
